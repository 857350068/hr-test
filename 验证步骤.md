# 快速验证步骤

## 第一步：检查数据库

### 1.1 连接MySQL并检查数据
```sql
-- 连接MySQL（在命令行执行）
mysql -u root -p

-- 检查数据库
SHOW DATABASES LIKE 'hr_db';

-- 使用数据库
USE hr_db;

-- 检查关键表的数据量
SELECT 'employee_profile' AS table_name, COUNT(*) AS count FROM employee_profile
UNION ALL
SELECT 'hr_department', COUNT(*) FROM hr_department
UNION ALL
SELECT 'sys_user', COUNT(*) FROM sys_user
UNION ALL
SELECT 'hr_data_category', COUNT(*) FROM hr_data_category;

-- 如果数据库不存在或表为空，执行初始化脚本
-- exit退出MySQL，然后执行：
-- mysql -u root -p < database/mysql/init.sql
```

### 1.2 预期结果
- employee_profile: 应该有几百条记录
- hr_department: 应该有17条记录
- sys_user: 应该有20条记录
- hr_data_category: 应该有8条记录

## 第二步：启动后端服务

### 2.1 检查后端配置

## 第五步：问题排查

### 5.1 如果图表不显示

#### 检查浏览器控制台
1. 按F12打开开发者工具
2. 查看Console标签页的错误信息
3. 查看Network标签页的API请求

#### 常见错误及解决方案

**错误1：Network Error / CORS Error**
```
原因：跨域问题
解决：检查后端CorsConfig.java配置
```

**错误2：401 Unauthorized**
```
原因：未登录或Token过期
解决：重新登录系统
```

**错误3：404 Not Found**
```
原因：API路径错误或后端未启动
解决：确认后端服务正常运行
```

**错误4：500 Internal Server Error**
```
原因：后端代码错误或数据库连接失败
解决：查看后端日志，检查数据库连接
```

**错误5：图表区域空白但无错误**
```
原因：数据为空
解决：检查数据库是否有数据，检查API返回的数据
```

### 5.2 调试技巧
