# 人力资源数据中心系统 - 用户功能使用指南

## 概述

本文档详细介绍了人力资源数据中心系统中从用户角度出发的实用功能，包括数据导出、数据导入、高级筛选、数据对比等功能的使用方法。

---

## 一、数据导出功能

### 1.1 功能介绍

系统支持将各类数据导出为Excel或CSV格式，方便用户进行离线分析和数据备份。

### 1.2 支持的导出数据类型

- **员工档案数据**：员工基本信息和分析数据
- **分析数据**：八大核心分析维度的数据
- **用户数据**：系统用户账号信息
- **部门数据**：组织架构部门信息
- **多维度数据**：批量导出多个分析维度的数据

### 1.3 导出格式

- **Excel格式**：`.xlsx`格式，支持复杂格式，适合数据分析
- **CSV格式**：`.csv`格式，纯文本格式，适合数据交换

### 1.4 使用方法

#### 方法一：快速导出

1. 在数据页面找到"导出数据"按钮
2. 点击下拉箭头，选择导出格式（Excel或CSV）
3. 系统自动下载文件

#### 方法二：配置导出

1. 点击"导出数据"按钮
2. 在弹出的对话框中选择：
   - 导出格式
   - 数据类型
   - 筛选条件（部门、时间周期、岗位等）
   - 文件名（可选）
3. 点击"开始导出"按钮

### 1.5 API接口

**导出员工档案为Excel**
```
GET /api/export/employee-profile/excel?deptId={部门ID}&period={周期}&job={岗位}
```

**导出分析数据为Excel**
```
GET /api/export/analysis-data/excel?categoryId={分类ID}&period={周期}
```

**导出报表为Excel**
```
GET /api/export/report/excel?templateId={模板ID}&parameters={参数}
```

**导出用户数据为Excel**
```
GET /api/export/users/excel
```

## 二、数据导入功能

### 2.1 功能介绍

系统支持通过Excel文件批量导入数据，包括员工档案、用户数据、部门数据等。

### 2.2 支持的导入数据类型

- **员工档案数据**：批量导入员工基本信息和分析数据
- **用户数据**：批量导入系统用户账号
- **部门数据**：批量导入组织架构部门
- **分析数据**：批量导入各维度的分析数据

### 2.3 使用方法

#### 步骤1：选择数据类型

1. 点击"导入数据"按钮
2. 在弹出的对话框中选择要导入的数据类型
3. 如需导入分析数据，还需选择数据分类

#### 步骤2：下载导入模板

1. 在对话框中点击"下载导入模板"按钮
2. 系统会下载对应数据类型的Excel模板文件
3. 按照模板格式填写数据

#### 步骤3：上传文件

1. 点击"下一步"进入文件上传页面
2. 拖拽Excel文件到上传区域，或点击选择文件
3. 系统只支持`.xlsx`或`.xls`格式的Excel文件

#### 步骤4：验证数据

1. 点击"验证数据"按钮
2. 系统会检查数据格式和内容
3. 如有错误，会显示详细的错误信息
4. 修正错误后重新验证

#### 步骤5：完成导入

1. 验证通过后，点击"开始导入"按钮
2. 系统会显示导入进度
3. 导入完成后显示导入结果（成功数量、失败数量、错误详情）

### 2.4 数据验证规则

**员工档案数据验证**
- 员工编号不能为空
- 姓名不能为空
- 数据分类不能为空
- 指标值不能为空
- 员工编号不能重复

**用户数据验证**
- 工号不能为空
- 姓名不能为空
- 角色不能为空
- 工号不能重复

**部门数据验证**
- 部门名称不能为空
- 部门名称不能重复

### 2.5 API接口

**导入员工档案数据**
```
POST /api/import/employee-profile
Content-Type: multipart/form-data
参数：file (Excel文件)
```

**导入用户数据**
```
POST /api/import/users
Content-Type: multipart/form-data
参数：file (Excel文件)
```

**验证导入数据**
```
POST /api/import/validate
Content-Type: multipart/form-data
参数：file (Excel文件), dataType (数据类型)
```

**下载导入模板**
```
GET /api/import/template/{dataType}
参数：dataType (employee_profile/user/department)
```

## 三、高级数据筛选功能

### 3.1 功能介绍

提供多条件组合的数据筛选功能，支持保存常用筛选条件，快速定位目标数据。

### 3.2 支持的筛选条件

- **部门筛选**：按部门筛选数据
- **数据分类筛选**：按八大核心分类筛选
- **时间周期筛选**：按月度周期筛选
- **时间范围筛选**：选择时间范围进行筛选
- **岗位筛选**：按岗位名称筛选（支持模糊搜索）
- **员工编号筛选**：按员工编号筛选（支持模糊搜索）
- **姓名筛选**：按姓名筛选（支持模糊搜索）
- **指标值范围筛选**：按指标值范围筛选

### 3.3 排序功能

- **排序字段**：创建时间、指标值、员工编号、姓名
- **排序方式**：升序、降序

### 3.4 使用方法

#### 基本筛选

1. 在数据页面找到"数据筛选"面板
2. 根据需要选择各个筛选条件
3. 筛选条件支持组合使用
4. 点击"查询"按钮应用筛选

#### 保存筛选条件

1. 设置好筛选条件后
2. 点击"保存筛选"按钮
3. 输入筛选条件名称
4. 筛选条件会保存到浏览器本地存储

#### 使用已保存的筛选

1. 在"已保存的筛选"区域
2. 点击要使用的筛选条件标签
3. 系统会自动应用该筛选条件

#### 重置筛选

1. 点击"重置"按钮
2. 所有筛选条件会恢复默认状态

### 3.5 API接口

**高级查询员工档案数据**
```
POST /api/query/employee-profile/advanced
Content-Type: application/json
Body: {
  "deptId": 1,
  "categoryId": 1,
  "period": "202401",
  "job": "销售经理",
  "employeeNo": "E001",
  "name": "张三",
  "valueMin": 80,
  "valueMax": 100,
  "sortBy": "create_time",
  "sortOrder": "DESC",
  "page": 1,
  "pageSize": 20
}
```

**获取筛选条件选项**
```
GET /api/query/filter-options/{filterType}
参数：filterType (dept/job/period/category)
```

**搜索数据**
```
GET /api/query/search?keyword={关键词}&categoryId={分类ID}&period={周期}
```

## 四、数据对比分析功能

### 4.1 功能介绍

支持对同一数据分类在不同时间周期的数据进行对比分析，帮助用户了解数据变化趋势。

### 4.2 对比功能特性

- **多维度对比**：支持八大核心分析维度的对比
- **时间周期对比**：选择任意两个时间周期进行对比
- **统计摘要**：自动计算平均变化率、提升部门数、下降部门数
- **可视化展示**：表格和图表双重展示对比结果
- **变化趋势**：显示每个部门的变化方向（上升/下降/不变）

### 4.3 使用方法

#### 执行数据对比

1. 进入数据对比分析页面
2. 选择要对比的数据分类
3. 选择第一个时间周期
4. 选择第二个时间周期
5. 点击"开始对比"按钮

#### 查看对比结果

**统计摘要卡片**
- 对比部门总数
- 平均变化率（带颜色标识）
- 提升部门数量
- 下降部门数量

**对比表格**
- 部门名称
- 周期1数据值
- 周期2数据值
- 变化率（百分比，带颜色标签）
- 变化趋势（图标显示）
- 变化幅度（进度条显示）

**对比图表**
- 柱状图展示两个周期的数据对比
- 不同颜色区分不同周期

#### 导出对比结果

1. 在对比结果页面
2. 点击"导出数据"按钮
3. 选择导出格式
4. 系统会导出包含筛选条件的数据

### 4.4 API接口

**查询分析数据对比**
```
GET /api/query/analysis-data/compare?categoryId={分类ID}&period1={周期1}&period2={周期2}
```

**查询数据趋势分析**
```
GET /api/query/analysis-data/trend?categoryId={分类ID}&startDate={开始日期}&endDate={结束日期}
```

**查询部门排名数据**
```
GET /api/query/analysis-data/ranking?categoryId={分类ID}&period={周期}&limit={数量}
```

## 五、数据汇总与统计功能

### 5.1 功能介绍

提供数据统计摘要和多维度数据汇总功能，帮助用户快速了解数据概况。

### 5.2 统计摘要功能

**统计指标**
- 总记录数
- 平均值
- 最小值
- 最大值
- 标准差
- 各等级数量和占比（优秀/良好/合格/不合格）

### 5.3 多维度汇总功能

- 支持同时汇总多个数据分类的数据
- 按部门、时间周期等维度进行汇总
- 提供多种汇总统计指标

### 5.4 使用方法

#### 查看统计摘要

1. 在数据分析页面
2. 选择数据分类和时间周期
3. 系统自动显示统计摘要卡片

#### 多维度汇总

1. 选择要汇总的数据分类（可多选）
2. 选择时间周期（可选）
3. 点击"查询汇总"按钮
4. 系统显示各分类的汇总数据

### 5.5 API接口

**查询数据统计摘要**
```
GET /api/query/analysis-data/summary-stats?categoryId={分类ID}&period={周期}
```

**查询多维度数据汇总**
```
GET /api/query/analysis-data/summary?categoryIds={分类ID列表}&period={周期}
```

## 七、常见问题解答

### Q1: 导出文件很大怎么办？

A: 可以使用筛选条件减少导出数据量，或者选择CSV格式减小文件大小。

### Q2: 导入数据时出现错误怎么办？

A: 请检查：
1. 是否使用了正确的导入模板
2. 数据格式是否符合要求
3. 是否有必填字段为空
4. 是否有重复数据

### Q3: 如何批量导出多个分析维度的数据？

A: 选择"多维度数据"作为导出类型，系统会在Excel中为每个维度创建一个Sheet。

### Q4: 保存的筛选条件在哪里？

A: 筛选条件保存在浏览器本地存储中，清除浏览器数据会丢失。

### Q5: 数据对比时为什么有些部门没有数据？

A: 可能是某些部门在某个时间周期没有数据，或者数据已被删除。

## 八、最佳实践建议

### 8.1 数据导出

1. 定期导出重要数据进行备份
2. 使用筛选条件只导出需要的数据
3. 为导出的文件添加有意义的文件名
4. Excel格式适合复杂分析，CSV格式适合系统间数据交换

### 8.2 数据导入

1. 始终使用官方提供的导入模板
2. 导入前先验证数据格式
3. 批量导入前先小批量测试
4. 保留原始数据文件，以便出错时重新导入

### 8.3 数据筛选

1. 保存常用的筛选条件，提高工作效率
2. 组合使用多个筛选条件精确查找数据
3. 使用排序功能快速定位目标数据
4. 定期清理不再需要的保存筛选条件

### 8.4 数据对比分析

1. 选择相邻的月份进行对比，观察短期趋势
2. 选择相同季度的不同年份进行对比，观察长期趋势
3. 关注变化率较大的部门，深入分析原因
4. 结合业务背景解读数据变化
