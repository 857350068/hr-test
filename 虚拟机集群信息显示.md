2. 深度分析：为什么你觉得“用不上”？
通过对比你上传的三个日志文件，我发现了一些“多余”的痕迹，这可能就是你觉得混乱的原因：
A. 潜在的“僵尸”组件：Zookeeper 和 HBase
证据： 在 bigdata1 的环境变量 PATH 中，包含了 /data/soft/apache-zookeeper-3.5.8-bin/bin 和 /data/soft/hbase-2.2.7/bin。
分析： 你的 jps 进程列表里虽然没有显示 HBase 和 Zookeeper 服务在跑，但环境变量里留有它们的路径。
建议： 完全不用管它们。 只要 jps 里没有 QuorumPeerMain (ZK) 或 HMaster (HBase)，它们就不会占用系统资源，只是磁盘上留了个文件夹而已。除非你的 Java 代码显式去连接它们，否则对你的项目无影响。
B. 架构上的“冗余”：SecondaryNameNode
证据： bigdata1 上运行着 15973 SecondaryNameNode。
分析： 在现代 Hadoop 2.x/3.x 的生产环境中，通常使用 HA (高可用) 模式，SecondaryNameNode 已经被淘汰了。但在学习/测试集群中，它常被用来做 NameNode 的冷备份。
建议： 这是 Hadoop 自身的组件，只要它在跑，就留着，不要动它，否则 HDFS 可能报错。
