## 4. 业务流程图

### 4.1 员工入职流程

```mermaid
sequenceDiagram
    participant HR as HR部门
    participant IT as IT部门
    participant System as HR系统
    participant DB as 数据库

    HR->>System: 创建员工账号
    System->>DB: 插入sys_user
    DB-->>System: 返回用户ID
    System-->>HR: 账号创建成功

    HR->>System: 录入员工信息
    System->>DB: 插入employee
    DB-->>System: 返回员工ID
    System-->>HR: 信息录入成功

    HR->>IT: 通知IT开通权限
    IT->>System: 配置系统权限
    System->>DB: 更新用户角色
    DB-->>System: 权限配置完成
    System-->>IT: 权限配置成功

    IT->>System: 分配办公设备
    System->>DB: 记录设备信息
    DB-->>System: 记录成功
    System-->>IT: 设备分配完成

    HR->>System: 安排入职培训
    System->>DB: 创建培训记录
    DB-->>System: 培训安排完成
    System-->>HR: 培训安排成功

    HR->>System: 完成入职流程
    System->>DB: 更新员工状态为在职
    DB-->>System: 状态更新成功
    System-->>HR: 入职流程完成
```

### 9.2 认证授权流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant Frontend as 前端
    participant Backend as 后端
    participant Auth as 认证服务
    participant Database as 数据库

    User->>Frontend: 输入用户名密码
    Frontend->>Backend: 登录请求
    Backend->>Auth: 验证用户凭证
    Auth->>Database: 查询用户信息
    Database-->>Auth: 返回用户数据
    Auth->>Auth: 验证密码(BCrypt)
    alt 验证失败
        Auth-->>Backend: 返回错误
        Backend-->>Frontend: 返回401
        Frontend-->>User: 登录失败
    else 验证成功
        Auth->>Auth: 生成JWT Token
        Auth-->>Backend: 返回Token
        Backend-->>Frontend: 返回Token
        Frontend->>Frontend: 存储Token
        Frontend-->>User: 登录成功
    end

    User->>Frontend: 访问受保护资源
    Frontend->>Backend: 携带Token请求
    Backend->>Auth: 验证Token
    Auth->>Auth: 检查Token有效性
    alt Token无效
        Auth-->>Backend: 返回错误
        Backend-->>Frontend: 返回401
        Frontend-->>User: 跳转登录
    else Token有效
        Auth->>Auth: 检查用户权限
        Auth-->>Backend: 返回权限信息
        Backend->>Backend: 验证资源访问权限
        alt 权限不足
            Backend-->>Frontend: 返回403
            Frontend-->>User: 权限不足
        else 权限通过
            Backend-->>Frontend: 返回数据
            Frontend-->>User: 显示数据
        end
    end
```
