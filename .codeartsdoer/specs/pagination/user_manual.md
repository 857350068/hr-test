# 分页功能用户手册

## 功能概述

分页功能为人力资源数据中心系统的各个管理模块提供了高效的数据浏览和查询能力。通过分页功能，用户可以：

- 快速浏览大量数据
- 灵活筛选查询条件
- 自定义每页显示条数
- 轻松翻页查看数据

**适用模块：**
- 部门管理
- 数据分类管理
- 预警规则管理
- 报表模板管理
- 我的收藏

---

## 1. 部门管理

### 1.1 访问部门管理页面

**操作步骤：**

1. 登录系统
2. 进入"系统管理"菜单
3. 点击"部门管理"

**页面布局：**

```
┌─────────────────────────────────────────────────────────┐
│ 部门管理                                    [新增部门] │
├─────────────────────────────────────────────────────────┤
│ 部门名称: [___________] [查询] [重置]                    │
├─────────────────────────────────────────────────────────┤
│ ┌─────────────────────────────────────────────────────┐ │
│ │ ID  │ 部门名称   │ 父部门  │ 排序  │ 操作          │ │
│ ├─────┼───────────┼────────┼───────┼───────────────┤ │
│ │ 1   │ 技术部     │ 顶级    │ 1     │ [编辑][删除]  │ │
│ │ 2   │ 市场部     │ 顶级    │ 2     │ [编辑][删除]  │ │
│ │ ... │ ...       │ ...    │ ...   │ ...           │ │
│ └─────────────────────────────────────────────────────┘ │
│                                                         │
│ 共 25 条  [1] 2 3 ...  每页 10 条 [▼]                 │
└─────────────────────────────────────────────────────────┘
```

### 1.2 分页操作

#### 1.2.1 翻页

**操作方式：**

1. **点击页码**：直接点击页码数字跳转到对应页面
2. **点击"上一页"**：返回上一页
3. **点击"下一页"**：跳转到下一页
4. **输入页码**：在页码输入框中输入页码，按回车键跳转

**示例：**
- 当前在第1页，点击"2"跳转到第2页
- 当前在第3页，点击"上一页"返回第2页

#### 1.2.2 修改每页条数

**操作方式：**

1. 点击"每页 10 条"下拉框
2. 选择需要的每页条数（10、20、50、100）
3. 系统自动刷新数据

**示例：**
- 从每页10条改为每页20条
- 数据列表将显示20条记录

### 1.3 筛选查询

#### 1.3.1 按部门名称查询

**操作步骤：**

1. 在"部门名称"输入框中输入关键词
2. 点击"查询"按钮
3. 系统显示名称包含关键词的部门

**示例：**
- 输入"技术"
- 显示"技术部"、"技术研发部"等名称包含"技术"的部门

#### 1.3.2 重置查询

**操作步骤：**

1. 点击"重置"按钮
2. 系统清空查询条件
3. 页码重置为第1页
4. 显示所有部门数据

### 1.4 数据操作

#### 1.4.1 新增部门

**操作步骤：**

1. 点击"新增部门"按钮
2. 在弹出的对话框中填写部门信息
3. 点击"确定"按钮
4. 系统自动刷新数据列表

#### 1.4.2 编辑部门

**操作步骤：**

1. 点击操作列的"编辑"按钮
2. 在弹出的对话框中修改部门信息
3. 点击"确定"按钮
4. 系统自动刷新数据列表

#### 1.4.3 删除部门

**操作步骤：**

1. 点击操作列的"删除"按钮
2. 在确认对话框中点击"确定"
3. 系统自动刷新数据列表

## 6. 常见问题

### 6.1 分页相关问题

**问题1：为什么页面显示"共 0 条"？**

**可能原因：**
1. 数据库中没有数据
2. 筛选条件过于严格，没有匹配的数据
3. 网络连接问题，数据加载失败

**解决方案：**
1. 检查数据库中是否有数据
2. 点击"重置"按钮，清除筛选条件
3. 刷新页面，重新加载数据
4. 检查网络连接

**问题2：为什么修改每页条数后数据没有变化？**

**可能原因：**
1. 当前页码超出新的分页范围
2. 网络连接问题，数据加载失败

**解决方案：**
1. 点击"重置"按钮，重新加载数据
2. 刷新页面，重新加载数据
3. 检查网络连接

**问题3：为什么翻页后数据重复？**

**可能原因：**
1. 浏览器缓存问题
2. 数据库数据重复

**解决方案：**
1. 清除浏览器缓存
2. 刷新页面，重新加载数据
3. 检查数据库数据

### 6.2 筛选相关问题

**问题1：为什么查询条件不生效？**

**可能原因：**
1. 查询条件输入错误
2. 数据库中没有匹配的数据
3. 网络连接问题，查询失败

**解决方案：**
1. 检查查询条件是否正确
2. 点击"重置"按钮，清除筛选条件，重新输入
3. 刷新页面，重新加载数据
4. 检查网络连接

**问题2：为什么重置后数据没有变化？**

**可能原因：**
1. 网络连接问题，数据加载失败
2. 浏览器缓存问题

**解决方案：**
1. 刷新页面，重新加载数据
2. 清除浏览器缓存
3. 检查网络连接

### 6.3 权限相关问题

**问题1：为什么无法访问某些页面？**

**可能原因：**
1. 用户权限不足
2. 用户未登录
3. 系统配置问题

**解决方案：**
1. 联系管理员，申请相应权限
2. 确认用户已登录

**可能原因：**

2. 数据被锁定
3. 系统维护中

**解决方案：**
1. 联系管理员，申请相应权限
2. 等待数据解锁
3. 等待系统维护完成

### 6.4 性能相关问题

**问题1：为什么页面加载很慢？**

**可能原因：**
1. 网络连接慢
2. 数据量大
3. 系统负载高

**解决方案：**
1. 检查网络连接
2. 减少每页显示条数

**解决方案：**

## 8. 最佳实践

### 8.1 分页使用建议

1. **合理设置每页条数**
   - 数据量少：使用10-20条
   - 数据量中等：使用20-50条
   - 数据量大：使用50-100条

2. **使用筛选条件**
   - 先使用筛选条件缩小数据范围
   - 再进行分页浏览
   - 提高查询效率

3. **避免频繁翻页**
   - 使用筛选条件定位目标数据
   - 减少不必要的翻页操作

### 8.2 筛选使用建议

1. **使用精确筛选**
   - 对于枚举类型（如规则类型、生效状态），使用下拉框精确选择
   - 对于文本类型（如名称），使用模糊查询

2. **组合使用筛选条件**
   - 可以同时使用多个筛选条件
   - 系统会返回同时满足所有条件的数据

3. **及时重置筛选条件**
   - 查询完成后，点击"重置"按钮
   - 清除筛选条件，恢复默认视图

### 8.3 数据操作建议

1. **操作前确认**
   - 删除数据前，仔细确认
   - 编辑数据前，备份重要信息

2. **操作后验证**
   - 操作完成后，检查数据是否正确
   - 如有异常，及时联系技术支持

3. **定期备份数据**
   - 重要数据定期备份
   - 避免数据丢失
