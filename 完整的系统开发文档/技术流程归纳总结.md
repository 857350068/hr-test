# 人力资源数据中心系统 - 完整技术流程归纳总结

> **项目概述**：人力资源数据中心系统是一个基于Spring Boot + Vue3构建的企业级HR数据分析平台，采用MySQL + Hive双数据库架构，支持组织效能分析、人才梯队建设、薪酬福利分析等八大核心功能模块。

### 1.3 双数据库架构说明

| 数据库 | 用途 | 存储内容 | 访问方式 |
|--------|------|----------|----------|
| **MySQL** | 业务数据库 | 用户信息、部门信息、员工档案、预警规则、报表模板等基础业务数据 | JDBC直连 |
| **Hive** | 分析数据库 | 员工分析数据、培训效果数据、部门绩效视图等分析型数据 | JDBC直连，HiveServer2端口10000 |

**数据流转**：MySQL存储基础业务数据 → 定时同步到Hive → Hive进行复杂分析查询 → 返回分析结果

### 4.1 Windows端打包

**1. 后端打包**：

### 4.3 虚拟机部署

**1. 创建部署目录**：

```bash
mkdir -p /opt/hr-deploy
cd /opt/hr-deploy

# 验证文件上传
ls -la
```

**2. 配置生产环境参数**：

```bash
# 创建生产配置文件
vim /opt/hr-deploy/application-prod.yml
```

```yaml
server:
  port: 8081

spring:
  datasource:
    type: com.alibaba.druid.pool.DruidDataSource
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/hr_db?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: 你的MySQL密码
    druid:
      initial-size: 5
      min-idle: 5
      max-active: 20
      max-wait: 60000
      validation-query: SELECT 1
      test-while-idle: true

  hive:
    url: jdbc:hive2://localhost:10000/default
    username: hadoop
    password: ""
    driver-class-name: org.apache.hive.jdbc.HiveDriver

logging:
  level:
    root: INFO
    com.hr.backend: INFO
  file:
    name: /opt/hr-deploy/backend.log
```

**3. 启动后端服务**：

```bash
cd /opt/hr-deploy

# 后台启动
nohup java -jar hr-datacenter-1.0.0.jar --spring.profiles.active=prod > backend.log 2>&1 &

# 记录进程ID
echo $! > backend.pid

# 查看日志
tail -f backend.log
```

**验证后端启动**：

```bash
# 检查进程
ps aux | grep hr-datacenter

# 测试接口
curl -s http://localhost:8081/api/auth/login -X POST \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}' | head -c 200
```

**4. 部署前端**：

```bash
# 复制前端文件到Nginx目录
mkdir -p /usr/share/nginx/html/hr
cp -r /opt/hr-deploy/dist/* /usr/share/nginx/html/hr/

# 验证文件
ls -la /usr/share/nginx/html/hr/
```

**5. 配置Nginx**：

```bash
# 创建Nginx配置文件
vim /etc/nginx/conf.d/hr-datacenter.conf
```

```nginx
server {
    listen 80;
    server_name 192.168.168.100 _;
    root /usr/share/nginx/html/hr;
    index index.html;

    # 前端SPA路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API反向代理到后端8081
    location /api {
        proxy_pass http://127.0.0.1:8081;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
```

**6. 重启Nginx**：

```bash
# 测试配置
nginx -t

# 重启Nginx
systemctl restart nginx

# 查看状态
systemctl status nginx
```

### 6.1 服务管理脚本

**创建启动脚本**：

```bash
vim /opt/hr-deploy/start.sh
```

```bash
#!/bin/bash

# 启动后端
cd /opt/hr-deploy
nohup java -jar hr-datacenter-1.0.0.jar --spring.profiles.active=prod > backend.log 2>&1 &

# 启动Nginx
systemctl start nginx

echo "服务启动完成"
```

**创建停止脚本**：

```bash
vim /opt/hr-deploy/stop.sh
```

```bash
#!/bin/bash

# 停止后端
if [ -f /opt/hr-deploy/backend.pid ]; then
    PID=$(cat /opt/hr-deploy/backend.pid)
    kill -9 $PID
    rm /opt/hr-deploy/backend.pid
fi

# 停止Nginx
systemctl stop nginx

echo "服务停止完成"
```

**创建重启脚本**：

```bash
vim /opt/hr-deploy/restart.sh
```

```bash
#!/bin/bash

/opt/hr-deploy/stop.sh
sleep 3
/opt/hr-deploy/start.sh
```

## 十、注意事项和风险点

### 10.1 开发环境风险

| 风险点 | 影响 | 预防措施 |
|--------|------|----------|
| JDK版本不匹配 | 编译失败 | 统一使用JDK 1.8 |
| 依赖冲突 | 运行时错误 | 定期更新依赖，排除冲突 |
| 数据库连接失败 | 无法启动 | 检查MySQL服务状态 |
| 端口占用 | 服务启动失败 | 检查端口使用情况 |

### 10.2 部署环境风险

| 风险点 | 影响 | 预防措施 |
|--------|------|----------|
| 网络不通 | 无法访问 | 配置正确的网络模式 |
| 防火墙限制 | 端口无法访问 | 开放必要端口 |
| 权限不足 | 文件无法操作 | 配置正确的文件权限 |
| 内存不足 | 服务崩溃 | 合理配置JVM参数 |

### 10.3 数据安全风险

| 风险点 | 影响 | 预防措施 |
|--------|------|----------|
| 数据丢失 | 业务中断 | 定期备份数据 |
| 密码泄露 | 安全风险 | 使用强密码，定期更换 |
| SQL注入 | 数据泄露 | 使用参数化查询 |
| XSS攻击 | 用户数据泄露 | 前端输入验证 |

### 10.4 性能风险

| 风险点 | 影响 | 预防措施 |
|--------|------|----------|
| 数据库慢查询 | 响应慢 | 优化SQL，添加索引 |
| 内存泄漏 | 服务崩溃 | 监控内存使用 |
| 连接池耗尽 | 无法访问 | 合理配置连接池 |
| 磁盘空间不足 | 服务异常 | 定期清理日志 |
