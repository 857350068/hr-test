## 一、需求分析

### 1.1 业务需求

#### 1.1.1 数据整合需求

**需求描述**：
企业人力资源数据分散在多个系统（ERP、OA、考勤系统、招聘平台等），缺乏统一的数据中心，导致数据孤岛现象严重，难以进行跨系统分析。

**业务目标**：
- 整合多源数据，建立统一的数据中心
- 实现数据标准化和数据治理
- 支持数据的实时采集和定期同步

**核心指标**：
- 整合数据源数量：≥5个
- 数据同步延迟：≤1小时
- 数据准确率：≥99%

#### 1.1.2 数据分析需求

**需求描述**：
传统关系型数据库在海量数据分析时性能低下，无法支持复杂的多维度关联分析，缺乏历史数据的深度挖掘能力。

**业务目标**：
- 提供多维度、多层次的数据分析能力
- 支持海量历史数据的快速查询
- 实现数据分析的可视化呈现

**核心指标**：
- 支持分析维度：≥10个
- 查询响应时间：≤3秒
- 图表数量：≥50个

#### 1.1.3 决策支持需求

**需求描述**：
人力资源决策主要依赖经验和直觉，缺乏量化的数据支撑，无法进行预测性分析。

**业务目标**：
- 提供可视化数据分析看板
- 建立预警模型，提前识别风险
- 支持多种报表导出和分享

**核心指标**：
- 看板数量：≥4个
- 预警类型：≥3类
- 报表种类：≥10种

#### 1.1.4 风险预警需求

**需求描述**：
缺乏实时的人力资源风险监控机制，无法及时发现员工流失、人才缺口、人力成本超支等风险。

**业务目标**：
- 建立实时风险监控机制
- 实现智能预警推送
- 支持预警历史追踪

**核心指标**：
- 预警准确率：≥85%
- 预警响应时间：≤10分钟
- 预警类型：≥3类

### 1.2 用户需求

**角色定位**：
负责全企业人力资源管理工作，需要全面掌握人力资源数据，进行数据管理和规则配置。

- 管理本部门培训效果
- 查看本部门预警信息

**优先级**：中

#### 1.2.3 企业管理层需求

**角色定位**：
负责企业战略决策，需要查看全局数据，分析趋势，制定战略。

**核心需求**：
- 查看全企业数据
- 分析人力资源趋势
- 监控关键指标
- 查看预警信息
- 查看战略报表

**优先级**：高

#### 1.2.4 后台管理员需求

**角色定位**：
负责系统运维和维护，需要管理系统配置、用户权限、数据维护。

### 2.2 核心功能详细设计

#### 2.2.1 用户认证与授权模块

##### 功能1：用户注册

**功能描述**：
新用户通过用户名、密码、邮箱等信息进行注册。

**输入**：
- 用户名（6-20位字母数字组合）
- 密码（8-16位，包含字母和数字）
- 确认密码
- 邮箱地址
- 手机号码（可选）
- 真实姓名
- 所属部门
- 角色（HR管理员/部门负责人/企业管理层）

**处理逻辑**：
1. 验证用户名格式和唯一性
2. 验证密码强度和一致性
3. 验证邮箱格式和唯一性
4. 验证手机号码格式（如果提供）
5. 将用户信息保存到数据库
6. 发送注册成功邮件

**输出**：
- 注册成功/失败提示
- 用户ID

**异常处理**：
- 用户名已存在：提示"用户名已存在"
- 邮箱已存在：提示"邮箱已存在"
- 密码不符合要求：提示"密码必须包含字母和数字，长度8-16位"
- 网络异常：提示"网络连接失败，请重试"

##### 功能2：用户登录

**功能描述**：
已注册用户通过用户名和密码进行登录。

**输入**：
- 用户名
- 密码
- 验证码（可选）

**处理逻辑**：
1. 验证用户名和密码
2. 验证验证码（如果启用）
3. 检查用户状态（是否禁用）
4. 生成JWT Token
5. 记录登录日志
6. 返回Token和用户信息

**输出**：
- JWT Token
- 用户信息（用户名、角色、权限）
- 登录成功/失败提示

**异常处理**：
- 用户名或密码错误：提示"用户名或密码错误"
- 用户被禁用：提示"账户已被禁用"
- 连续5次登录失败：锁定账户30分钟
- 验证码错误：提示"验证码错误"

##### 功能3：权限管理

**功能描述**：
为用户分配系统角色，控制用户访问权限。

**输入**：
- 用户ID
- 角色ID

**处理逻辑**：
1. 验证用户和角色存在性
2. 更新用户角色
3. 清除用户权限缓存
4. 记录权限变更日志

**输出**：
- 权限分配成功/失败提示

**权限矩阵**：

| 功能模块 | HR管理员 | 部门负责人 | 企业管理层 | 后台管理员 |
|---------|---------|-----------|-----------|-----------|
| 用户管理 | ✗ | ✗ | ✗ | ✓ |
| 数据管理 | ✓ | ✗ | ✗ | ✓ |
| 规则管理 | ✓ | ✗ | ✗ | ✓ |
| 模型管理 | ✓ | ✗ | ✗ | ✓ |
| 报表管理 | ✓ | ✗ | ✓ | ✓ |
| 数据查看(全企业) | ✓ | ✗ | ✓ | ✓ |
| 数据查看(本部门) | ✓ | ✓ | ✗ | ✓ |
| 预警查看 | ✓ | ✓ | ✓ | ✓ |

#### 2.2.2 数据展示与查询模块

##### 功能1：数据分类展示

**功能描述**：
以分类形式展示人力资源数据，包括组织效能类、人才梯队类、薪酬福利类、绩效管理类等。

**输入**：
- 无（默认展示所有分类）

**处理逻辑**：
1. 查询所有数据分类
2. 统计每个分类的数据条数
3. 返回分类列表

**输出**：
- 分类ID
- 分类名称
- 分类描述
- 分类图标
- 数据条数
- 创建时间

**数据分类**：
1. 组织效能类
2. 人才梯队类
3. 薪酬福利类
4. 绩效管理类
5. 培训效果类
6. 人力成本类
7. 员工发展类
8. 预警监控类

##### 功能2：数据列表展示

**功能描述**：
默认展示所有人力资源核心分析数据，包括数据维度、统计周期、核心指标数值、趋势图表等。

**输入**：
- 分类ID（可选，默认全部）
- 页码（默认1）
- 每页条数（默认20）

**处理逻辑**：
1. 根据分类ID筛选数据
2. 查询数据列表
3. 统计总数
4. 分页返回

**输出**：
- 数据ID
- 数据维度
- 统计周期
- 核心指标数值
- 趋势变化率
- 趋势图表数据
- 创建时间
- 分类名称

**列表字段**：
- 数据维度：如"人均产值"、"人员利用率"等
- 统计周期：如"2024-Q1"、"2024-01"等
- 核心指标数值：如"85.5"、"92.3"等
- 趋势变化率：如"+5.2%"、"-3.1%"等
- 趋势图表：ECharts图表配置

##### 功能3：模糊查询

**功能描述**：
支持根据部门、岗位、统计周期、员工编号进行模糊查询。

**输入**：

- 岗位名称（可选）

2. 执行模糊查询

**输出**：
- 符合条件的数据列表
- 查询结果总数

**查询规则**：
- 多个条件为AND关系
- 支持部分匹配
- 不区分大小写

支持将查询结果导出为Excel文件。

**输入**：

- 数据维度
- 统计周期

- 趋势变化率
- 备注

#### 2.2.4 数据分析模块

##### 功能1：组织效能分析

**功能描述**：
查看组织效能分析数据，包括人均产值、人员利用率、组织架构合理性等指标。

**输入**：
- 部门ID（可选）
- 时间范围

**处理逻辑**：
1. 查询组织效能基础数据
2. 计算组织效能指标
3. 生成分析报告
4. 返回分析结果

**输出**：
- 人均产值
- 人员利用率
- 组织架构合理性
- 组织效能指数
- 趋势分析
- 对比分析

**核心指标**：

| 指标名称 | 计算公式 | 说明 |
|---------|---------|------|
| 人均产值 | 总产值/员工人数 | 衡量员工产出效率 |
| 人员利用率 | 实际工作时间/应工作时间 | 衡量人员利用程度 |
| 组织架构合理性 | （管理层级数+部门数）/员工人数 | 衡量组织结构合理性 |
| 组织效能指数 | 综合评分 | 综合评估组织效能 |

##### 功能2：人才梯队分析

**功能描述**：
查看人才梯队分析数据，包括各层级人才分布、关键岗位储备率、继任计划覆盖率等。

**输入**：
- 部门ID（可选）
| 关键岗位储备率 | 继任计划覆盖率 | 衡量人才储备情况 |
| 继任计划覆盖率 | 有继任者岗位数/关键岗位总数 | 衡量继任计划完善度 |

**输出**：
- 各层级人才分布
- 关键岗位储备率
- 继任计划覆盖率
- 人才梯队健康度
- 人才缺口分析

## 三、数据流程设计

### 3.1 用户登录流程

```
用户输入用户名和密码
    ↓
前端验证输入格式
    ↓
发送登录请求到后端
    ↓
后端验证用户名和密码
    ↓
生成JWT Token
    ↓
返回Token和用户信息
    ↓
前端存储Token
    ↓
跳转到数据看板页面
```

### 3.2 数据查询流程

```
用户选择查询条件
    ↓
前端验证查询条件
    ↓
发送查询请求到后端
    ↓
后端验证用户权限
    ↓
查询数据库（MySQL或Hive）
    ↓
返回查询结果
    ↓
前端展示数据
    ↓
生成图表（ECharts）
```

### 3.3 数据同步流程

```
定时任务触发
    ↓
从MySQL读取基础数据
    ↓
数据清洗和转换
    ↓
写入Hive
    ↓
记录同步日志
    ↓
更新同步状态
```

### 3.4 预警触发流程

```
预警引擎定时检测
    ↓
读取预警规则
    ↓
检测数据是否符合预警条件
    ↓
触发预警
    ↓
生成预警信息
    ↓
发送预警通知
    ↓
记录预警日志
```

## 五、总结

本文档详细描述了人力资源数据中心系统的需求分析和功能设计，包括业务需求、用户需求、非功能性需求，以及各个功能模块的详细设计。

系统采用前后端分离的架构，前端使用Vue 3 + Element Plus，后端使用Spring Boot + MyBatis Plus，数据库采用MySQL + Hive双数据库架构。

系统功能完善，涵盖了用户认证、数据展示、数据分析、预警管理、培训管理、绩效管理、收藏管理、用户管理、数据管理、规则管理、模型管理、报表管理等核心功能。

系统具有良好的可扩展性和可维护性，为后续的功能扩展和性能优化提供了坚实的基础。
