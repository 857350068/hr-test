## 一、数据库概述

### 1.1 双数据库架构

本系统采用MySQL + Hive双数据库架构，充分发挥两种数据库的优势：

| 数据库 | 用途 | 特点 | 适用场景 |
|--------|------|------|----------|
| **MySQL** | 业务数据库 | ACID事务、索引优化、响应快 | 用户管理、基础数据、业务操作 |
| **Hive** | 分析数据库 | 分布式存储、海量数据、复杂查询 | 数据分析、报表生成、历史数据 |

### 1.2 数据流转

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│  业务操作   │ ───→ │   MySQL     │ ───→ │   Hive      │
│ (增删改查)  │      │ (业务数据)   │      │ (分析数据)   │
└─────────────┘      └─────────────┘      └─────────────┘
                           ↑                     ↑
                           │                     │
                      实时写入              定时同步
                           │                     │
                           └─────────────┬───────┘
                                         ↓
                                  数据分析查询
```

### 1.3 数据库版本

| 数据库 | 版本 | 字符集 | 排序规则 |
|--------|------|--------|----------|
| MySQL | 8.0.33 | utf8mb4 | utf8mb4_unicode_ci |
| Hive | 3.1.3 | utf8 | - |

| 字段名 | 类型 | 长度 | 允许空 | 默认值 | 说明 |
|--------|------|------|--------|--------|------|

## 四、数据同步设计

### 4.1 同步策略

**同步类型**:
- 全量同步：每日凌晨2点执行
- 增量同步：每小时执行一次

**同步方向**: MySQL → Hive

### 4.2 同步流程

```
1. 从MySQL读取增量数据
   ↓
2. 数据清洗和转换
   ↓
3. 写入Hive临时表
   ↓
4. 数据验证
   ↓
5. 合并到正式表
   ↓
6. 记录同步日志
```

### 4.3 同步监控

**监控指标**:
- 同步状态（成功/失败）
- 同步记录数
- 同步耗时
- 错误日志

## 五、数据库优化

### 5.1 MySQL优化

1. **索引优化**
   - 为常用查询字段创建索引
   - 避免过度索引
   - 定期分析索引使用情况

2. **查询优化**
   - 使用EXPLAIN分析查询计划
   - 避免全表扫描
   - 优化JOIN操作

3. **连接池优化**
   - 合理设置连接池大小
   - 配置连接超时时间
   - 监控连接池使用情况

### 5.2 Hive优化

1. **分区优化**
   - 按时间维度分区
   - 合理设置分区数量
   - 定期清理过期分区

2. **存储格式优化**
   - 使用ORC列式存储
   - 启用压缩
   - 合理设置块大小

3. **查询优化**
   - 使用分区裁剪
   - 避免全表扫描
   - 优化JOIN操作

## 六、数据库备份与恢复

### 6.1 MySQL备份

**备份策略**:
- 每日全量备份
- 每小时增量备份
- 保留30天备份

**备份命令**:
```bash
# 全量备份
mysqldump -u root -p hr_db > hr_db_$(date +%Y%m%d).sql

# 增量备份（使用binlog）
mysqlbinlog --start-datetime="2024-01-01 00:00:00" --stop-datetime="2024-01-01 23:59:59" mysql-bin.000001 > incremental.sql
```

### 6.2 Hive备份

**备份策略**:
- 每周全量备份
- 每日增量备份
- 保留90天备份

**备份命令**:
```bash
# 全量备份（导出HDFS数据）
hdfs dfs -get /user/hive/warehouse/hr_warehouse /backup/hr_warehouse_$(date +%Y%m%d)

# 增量备份（导出分区数据）
hdfs dfs -get /user/hive/warehouse/hr_warehouse/dw_org_efficiency/year=2024/month=01 /backup/dw_org_efficiency_202401
```

### 6.3 恢复

**MySQL恢复**:
```bash
# 恢复全量备份
mysql -u root -p hr_db < hr_db_20240101.sql

# 恢复增量备份
mysql -u root -p hr_db < incremental.sql
```

**Hive恢复**:
```bash
# 恢复全量备份
hdfs dfs -put /backup/hr_warehouse_20240101 /user/hive/warehouse/hr_warehouse

# 恢复分区
hdfs dfs -put /backup/dw_org_efficiency_202401 /user/hive/warehouse/hr_warehouse/dw_org_efficiency/
```

## 七、总结

本文档详细描述了人力资源数据中心系统的数据库设计，包括MySQL和Hive双数据库架构的表结构设计、数据同步机制、数据库优化策略以及备份恢复方案。

MySQL作为业务数据库，存储用户信息、部门信息、员工档案、预警规则等基础业务数据，支持高效的OLTP操作。

Hive作为分析数据库，存储组织效能、人才梯队、薪酬福利、绩效管理、培训效果等分析数据，支持海量数据的OLAP操作。

系统采用定时同步机制，将MySQL的基础数据同步到Hive，保证数据的一致性和完整性，为数据分析提供可靠的数据支撑。
