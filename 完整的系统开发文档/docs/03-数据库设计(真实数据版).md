## 一、数据库概述

### 1.1 双数据库架构

本系统采用MySQL + Hive双数据库架构，充分发挥两种数据库的优势：

| 数据库 | 用途 | 特点 | 适用场景 |
|--------|------|------|----------|
| **MySQL** | 业务数据库 | ACID事务、索引优化、响应快 | 用户管理、基础数据、业务操作 |
| **Hive** | 分析数据库 | 分布式存储、海量数据、复杂查询 | 数据分析、报表生成、历史数据 |

### 1.2 数据库版本

| 数据库 | 版本 | 字符集 | 排序规则 |
|--------|------|--------|----------|
| MySQL | 8.0.33 | utf8mb4 | utf8mb4_unicode_ci |
| Hive | 3.1.3 | utf8 | - |

### 2.2 表结构设计

#### 2.2.1 用户表（sys_user）

**表说明**: 存储系统用户信息

| 字段名 | 类型 | 长度 | 允许空 | 默认值 | 说明 |
|--------|------|------|--------|--------|------|
| id | BIGINT | 20 | 否 | - | 用户ID（主键） |
| username | VARCHAR | 50 | 否 | - | 用户名（唯一） |
| password | VARCHAR | 255 | 否 | - | 密码（BCrypt加密） |
| real_name | VARCHAR | 50 | 否 | - | 真实姓名 |
| email | VARCHAR | 100 | 否 | - | 邮箱（唯一） |
| phone | VARCHAR | 20 | 是 | NULL | 手机号码 |
| role_id | BIGINT | 20 | 否 | - | 角色ID（外键） |
| department_id | BIGINT | 20 | 是 | NULL | 部门ID（外键） |
| status | TINYINT | 1 | 否 | 1 | 状态（1-正常，0-禁用） |
| avatar | VARCHAR | 255 | 是 | NULL | 头像URL |
| last_login_time | DATETIME | - | 是 | NULL | 最后登录时间 |
| last_login_ip | VARCHAR | 50 | 是 | NULL | 最后登录IP |
| create_time | DATETIME | - | 否 | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | - | 否 | CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |
| create_by | VARCHAR | 50 | 是 | NULL | 创建人 |
| update_by | VARCHAR | 50 | 是 | NULL | 更新人 |
| remark | VARCHAR | 500 | 是 | NULL | 备注 |

## 四、数据特点说明

### 4.1 真实性改进

相比之前的测试数据,本次数据改进如下:

1. **企业背景真实**: 采用腾讯科技为背景,使用真实的部门名称和岗位名称
2. **薪酬水平真实**: 参考真实互联网大厂的薪酬水平,从1.5万到13万不等
3. **绩效评分真实**: S/A/B/C/D分级,分数从78到96,符合实际绩效分布
4. **培训内容真实**: 包含真实的技术培训和管理培训课程
5. **预警场景真实**: 基于真实业务场景的预警信息

### 4.2 数据规模

- MySQL表: 13张表,每表10-15条数据
- Hive表: 5张表,每表10条数据
- 总数据量: 约150条记录

### 4.3 数据关联

- 用户→部门→岗位→员工: 完整的关联关系
- 员工→绩效记录→薪酬记录: 业务数据关联
- MySQL→Hive: 数据同步关系
