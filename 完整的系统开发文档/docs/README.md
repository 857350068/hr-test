## 文档使用指南

### 开发人员

**推荐阅读顺序**:
1. 项目概述与技术架构 → 了解整体架构
2. 需求分析与功能设计 → 了解功能需求
3. 数据库设计 → 了解数据结构
4. 前端开发指南/后端开发指南 → 开始开发

**推荐阅读顺序**:
1. 项目概述与技术架构 → 了解系统架构
2. 部署运维指南 → 部署系统
3. API接口文档 → 了解系统接口
4. 测试文档 → 进行系统测试

### 项目经理

2. 需求分析与功能设计 → 了解功能需求
3. 开发任务书 → 了解任务分配
4. 测试文档 → 了解测试情况
