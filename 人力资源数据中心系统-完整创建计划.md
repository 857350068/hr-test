# 人力资源数据中心系统 - 完整创建计划

## 一、文档依据与目标

- **依据**：`完整的系统开发文档/人力资源数据中心系统-完整开发文档.md`、`任务书和开题报告.md`
- **目标**：在 Windows 11 主机完成前后端与双库设计开发，打包部署到 CentOS 7 虚拟机（IP：192.168.168.100），通过主机浏览器访问 `http://192.168.168.100:端口` 使用完整服务。
- **双数据库**：MySQL 8.0.33（业务数据）+ Hive 3.1.3（分析数据），各提供完整建表 SQL 及不少于 10 条初始数据。

---

## 二、技术栈与环境

| 层级     | 技术/版本        | 说明 |
|----------|------------------|------|
| 前端     | Vue 3、Vite 5、Element Plus、ECharts、Pinia、Axios | 单页应用，REST 调用后端 |
| 后端     | Spring Boot 2.7.18、MyBatis Plus、JWT、Druid、EasyExcel | RESTful API，双数据源 |
| 业务库   | MySQL 8.0.33     | 用户、部门、分类、档案、规则、报表、收藏、同步日志、操作日志 |
| 分析库   | Hive 3.1.3 + Hadoop | 分析表、培训效果表、视图，MySQL→Hive 同步 |
| 开发环境 | Windows 11、JDK 1.8、Maven、Node.js | 无本地 Hadoop/Hive，仅连虚拟机 Hive |
| 部署环境 | CentOS 7 虚拟机 | JDK 1.8、MySQL 8、Hadoop、Hive，IP 192.168.168.100 |

## 三、完整创建计划总览

### 阶段一：数据库与脚本（先行）

1. **MySQL**
   - 建库 `hr_db`，建表：sys_user、hr_department、hr_data_category、employee_profile、warning_rule、report_template、sys_favorite、data_sync_log、sys_log。
   - 每张业务表插入不少于 10 条合理初始数据（用户、部门、分类、档案、规则、报表模板、收藏、同步日志、操作日志）。
2. **Hive**
   - 建库/默认库下建表：hr_analytic_data、hr_training_effect；建视图 vw_dept_performance、vw_training_summary。
   - 插入不少于 10 条分析数据与培训数据，便于前端图表展示。
3. **交付物**
   - `database/mysql/init.sql`：MySQL 完整脚本。
   - `database/hive/init.sql`：Hive 建表+视图+数据脚本。

### 阶段二：后端（Spring Boot）

4. **项目骨架**
   - 包结构：controller、service、mapper、model(entity/dto)、config、security、exception、util。
   - 配置文件：application.yml（公共）、application-dev.yml（Windows 连本机 MySQL + 虚拟机 Hive）、application-prod.yml（CentOS 连本机 MySQL + 本机 Hive）。
5. **双数据源**
   - 主数据源：MySQL（Druid），用于所有业务表。
   - 次数据源：Hive（JDBC），用于分析查询与数据同步写入；无 Hive 时分析接口可回退到 MySQL 查询。
6. **功能模块**
   - **认证**：登录、注册、获取用户信息、登出、修改密码；JWT 签发与校验；BCrypt 密码。
   - **用户与权限**：角色 HR_ADMIN / DEPT_HEAD / MANAGEMENT / EMPLOYEE；部门范围权限（dept_scope）。
   - **数据分类**：树形展示，增删改查（hr_data_category）。
   - **员工档案**：分页、条件查询、增删改、Excel 导入导出（EasyExcel）。
   - **八大分析接口**：组织效能、人才梯队、薪酬福利、绩效管理、员工流失预警、培训效果、人力成本优化、人才发展预测；支持 period 等参数，优先读 Hive，不可用时读 MySQL。
   - **预警规则**：增删改查、启用/禁用、生效规则不可删仅可记日志。
   - **报表模板**：增删改查、预览参数。
   - **收藏**：按用户增删改查（报表/预警/人才）。
   - **数据同步**：MySQL → Hive 同步任务，记录 data_sync_log；支持手动触发。
   - **管理员**：用户管理（CRUD、状态、重置密码）、数据管理、数据分类管理、规则管理、报表管理。
7. **安全与异常**
   - JWT Filter、Spring Security 配置、CORS。
   - 全局异常处理、统一响应码、参数校验。

### 阶段三：前端（Vue 3）

8. **项目骨架**
   - Vite + Vue 3 + Element Plus + ECharts + Pinia + Vue Router。
   - 目录：api、assets、components、router、stores、views（含 analysis、admin）、Layout。
9. **基础能力**
   - request 封装（baseURL、Authorization、401 跳转登录）。
   - 路由守卫：登录校验、管理员校验；标题与 403/404。
   - User Store：token、userInfo、isAdmin、login/logout/getUserInfo。
10. **页面与 UI**
    - 登录、注册；主布局（侧栏+顶栏+内容区）。
    - 仪表盘：统计卡片、数据分类入口、预警列表、快捷入口（参考文档 7 章 UI 规范）。
    - 八大分析页：各对应一个分析接口，ECharts 图表 + 表格 + 筛选（如 period）。
    - 数据分析总览：分类列表与跳转。
    - 个人中心、我的收藏、数据导入导出（上传/下载/模板）。
    - 管理员：用户管理、数据分类管理、数据管理、预警规则管理、报表管理、数据同步触发。
    - 403/404 页。
11. **与文档一致**
    - 颜色、字体、卡片风格按《人力资源数据中心系统-完整开发文档》第七章 UI 设计规范执行。

### 阶段四：联调与部署

12. **联调**
    - 主机前端 dev 代理到主机后端；主机后端连主机 MySQL + 虚拟机 Hive（url 中 IP 为 192.168.168.100 或实际虚拟机 IP）。
    - 验证：登录、八大分析、档案 CRUD、规则、报表、收藏、同步。
13. **部署**
    - 前端打包：`npm run build`，将 dist 交给 Nginx 或后端静态资源。
    - 后端打包：`mvn package`，得到 jar。
    - 虚拟机：安装 JDK 1.8、MySQL 8、Hadoop、Hive；执行 database/mysql/init.sql、database/hive/init.sql；上传 jar 与前端资源；使用 application-prod.yml（MySQL/Hive 改为本机或 localhost）；启动后端；配置 Nginx 或直接访问后端端口。
14. **访问**
    - 主机浏览器：`http://192.168.168.100:后端端口` 或 `http://192.168.168.100:80`（若 Nginx 反向代理）。

## 四、数据库设计要点（与文档一致）

- **MySQL**：sys_user（username 唯一）、hr_department、hr_data_category、employee_profile（含 category_id、period）、warning_rule（rule_type、effective、update_log）、report_template、sys_favorite（fav_type）、data_sync_log、sys_log。
- **Hive**：hr_analytic_data（与 employee_profile 字段对应，便于同步）、hr_training_effect；视图按文档 4.3.3。
- **同步**：由后端定时或手动从 MySQL 读出 employee_profile，写入 Hive hr_analytic_data，并写 data_sync_log。

## 八、开发顺序建议

1. 建 MySQL 库表并执行 init.sql，确认数据正确。
2. 在虚拟机建 Hive 表与视图并执行 Hive init.sql。
3. 搭建后端工程：配置、实体、Mapper、双数据源、Auth 与 JWT。
4. 实现后端各 Controller/Service（先认证与八大分析，再档案、规则、报表、收藏、同步）。
5. 搭建前端工程：request、router、store、Layout、登录注册。
6. 实现仪表盘与八大分析页，再个人中心、收藏、导入导出。
7. 实现管理员各页与数据同步。
8. 联调、打包、部署到虚拟机，主机浏览器访问验证。
