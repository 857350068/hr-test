# 项目结构整理说明

## 整理前的问题
项目原本存在以下目录重复问题：
1. `backend` 和 `hr-backend` - 两个后端项目
2. `frontend` 和 `hr-frontend` - 两个前端项目
3. `deploy` 和 `hr-docs` - 空目录

## 整理方案
采用**方案A：保留新版，归档旧版**，具体操作如下：

### 1. 目录重命名
- `backend` → `backend-legacy` (旧版数据分析中心后端)
- `frontend` → `frontend-legacy` (旧版数据分析中心前端)
- `hr-backend` → `backend` (新版人力资源管理系统后端)
- `hr-frontend` → `frontend` (新版人力资源管理系统前端)

### 2. 空目录清理
- 删除 `deploy` 目录（空目录）
- 删除 `hr-docs` 目录（空目录）

### 3. 配置文件更新
- 更新 `start.bat` 启动脚本中的路径引用
- 更新 `README.md` 文档中的路径说明

## 功能对比

### 新版 (backend/frontend) - 人力资源管理系统
- **包名**: `com.hr.datacenter`
- **功能**: 员工管理、考勤、绩效、薪酬、培训等HR核心功能
- **技术栈**: Spring Boot + MyBatis Plus + JWT + Vue 3 + Element Plus
- **状态**: 当前活跃版本，启动脚本和文档都指向此版本

### 旧版 (backend-legacy/frontend-legacy) - 数据分析中心
- **包名**: `com.hr.backend`
- **功能**: 数据查询、分析、报表、系统管理等数据分析功能
- **技术栈**: Spring Boot + MyBatis Plus + Hive + Swagger + Vue 3
- **状态**: 历史版本，已归档为legacy

## 使用说明

### 启动项目
```bash
# 使用启动脚本
start.bat

# 或手动启动
cd backend
mvn spring-boot:run

cd ../frontend
npm run dev
```

### 数据库初始化
```bash
mysql -u root -p
source backend/src/main/resources/sql/init.sql
```
