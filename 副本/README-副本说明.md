# HrDataCenter 代码副本说明

这个目录用于保存当前项目的“可重建基线”，方便后续从头重构。

## 目录结构

- `前端`：从 `frontend` 提取的源码与构建配置（不含 `dist`、`node_modules`）。
- `后端`：从 `backend` 提取的业务源码与Maven配置（不含 `target`）。
- `数据库`：从 `database` 及根目录测试SQL提取的脚本集合。
- `技术文档`：重构需要的技术档案与功能模块清单。

## 使用建议

1. 先阅读 `技术文档/01-重构技术档案.md`，确定技术栈与版本。
2. 再阅读 `技术文档/02-功能模块全量清单.md`，按模块重建前后端与数据库。
3. 按 `技术文档/03-从零重建执行清单.md` 逐步执行启动流程。
4. 需要虚拟机整站部署时，按 `技术文档/04-虚拟机整站部署与IP访问.md` 执行。
5. 本副本里的 `后端/application.yml.example` 已做脱敏，重建时请自行填入环境变量。
