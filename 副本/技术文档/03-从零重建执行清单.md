# HrDataCenter 从零重建执行清单（可直接启动）

## 0. 使用方式

- 目标：按本清单从空环境启动“数据库 + 后端 + 前端”，并完成基础联调验收。
- 适用系统：Windows 10/11（PowerShell）。
- 执行目录基线：`D:/HrDataCenter/副本`。

---

## 1. 环境准备清单（先检查，再动手）

### 1.1 必需软件

- JDK 8（建议 `1.8.0_281+`）
- Maven 3.6+
- Node.js 16 或 18（建议 `18 LTS`）
- MySQL 8.0+
- Hive 3.1+（如果你先只跑业务模块，可先跳过 Hive）

### 1.2 一键检查命令（PowerShell）

```powershell
java -version
mvn -version
node -v
npm -v
mysql --version
```

### 1.3 结果判定

- `java version "1.8..."` 或 `openjdk version "1.8..."`：通过
- Maven 可正常输出版本：通过
- Node 与 npm 可正常输出版本：通过
- MySQL 客户端可输出版本：通过

## 2. 数据库初始化（MySQL + Hive）

## 2.1 MySQL 初始化

### 2.1.1 建库并导入结构

```powershell
mysql -uroot -p -e "CREATE DATABASE IF NOT EXISTS hr_datacenter DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;"
mysql -uroot -p hr_datacenter < "D:/HrDataCenter/副本/数据库/hr_datacenter_mysql_init.sql"
```

### 2.1.2 导入基础数据（可选）

```powershell
mysql -uroot -p hr_datacenter < "D:/HrDataCenter/副本/数据库/1mysql/insert_data.sql"
```

### 2.1.3 导入大样本数据（性能/压测场景可选）

```powershell
mysql -uroot -p hr_datacenter < "D:/HrDataCenter/副本/数据库/1mysql/insert_large_data.sql"
```

### 2.1.4 执行补丁脚本（建议执行）

```powershell
mysql -uroot -p hr_datacenter < "D:/HrDataCenter/副本/数据库/mysql_patch_20260416.sql"
mysql -uroot -p hr_datacenter < "D:/HrDataCenter/副本/数据库/update_user_passwords.sql"
```

## 2.2 Hive 初始化（分析模块需要）

```powershell
hive -f "D:/HrDataCenter/副本/数据库/hr_datacenter_hive_init.sql"
hive -f "D:/HrDataCenter/副本/数据库/2hive/insert_data.sql"
```

如需更大数据量：

```powershell
hive -f "D:/HrDataCenter/副本/数据库/2hive/insert_large_data.sql"
```

## 3. 后端启动（Spring Boot）

## 3.1 配置环境变量（当前 PowerShell 会话）

> `application.yml.example` 已脱敏，建议用环境变量注入敏感配置。

```powershell
$env:MYSQL_PASSWORD = "你的MySQL密码"
$env:JWT_SECRET = "请替换成长度足够的JWT密钥_建议64字节以上"
```

## 3.2 检查并准备配置文件

- 配置示例文件：`副本/后端/application.yml.example`
- 建议复制为运行配置文件：

```powershell
Copy-Item "D:/HrDataCenter/副本/后端/application.yml.example" "D:/HrDataCenter/副本/后端/src/main/resources/application.yml" -Force
```

- 关键检查项：
  - MySQL 地址、端口、库名是否正确
  - Hive 地址是否可达（若暂时不用分析，可先保留但不调用分析页面）
  - `server.servlet.context-path` 为 `/api`

## 3.3 编译与启动

```powershell
Set-Location "D:/HrDataCenter/副本/后端"
mvn clean package -DskipTests
mvn spring-boot:run
```

## 3.4 启动成功标志

- 控制台出现 `Started ...Application`
- 服务可访问：`http://localhost:8080/api`

---

## 4. 前端启动（Vue3 + Vite）

## 4.1 安装依赖

```powershell
Set-Location "D:/HrDataCenter/副本/前端"
npm install
```

## 4.2 启动开发服务

```powershell
npm run dev
```

## 4.3 启动成功标志

- 控制台出现 Vite 本地地址（例如 `http://localhost:5173`）

## 5. 联调配置核对（最关键）

## 5.1 前端请求前缀

- 检查前端请求配置文件（通常在 `前端/src/api/request.js`）：
  - 开发环境 baseURL 应指向 `http://localhost:8080/api`

## 5.2 登录联调

- 打开前端登录页
- 输入数据库中可用账号
- 登录后应跳转到仪表盘/首页

## 5.3 路由权限联调

- 普通员工账号：只能访问基础业务页
- 管理员账号：可访问系统管理页（用户、日志、规则、模型、报表）

## 7. 常见故障与处理

## 7.1 后端启动失败：数据库连接异常

- 检查 MySQL 是否启动
- 检查 `jdbc-url` 主机与端口
- 检查 `$env:MYSQL_PASSWORD` 是否正确

## 7.2 后端启动失败：JWT 相关异常

- 检查 `$env:JWT_SECRET` 是否设置
- 密钥长度建议足够（尽量 64 字节以上）

## 7.3 前端启动失败：依赖安装报错

- 删除 `前端/node_modules` 后重装：

```powershell
Remove-Item "D:/HrDataCenter/副本/前端/node_modules" -Recurse -Force
npm install
```

## 7.4 前端白屏/接口 404

- 确认后端是否在 `8080` 端口运行
- 确认后端上下文路径是否为 `/api`
- 确认前端 `baseURL` 与后端一致

## 7.5 分析页面无数据

- 检查 Hive 初始化是否已执行
- 检查 Hive 地址可达性
- 先验证 MySQL 业务页面是否正常，分步排错
