## 目标

- 所有服务运行在虚拟机（MySQL/Hive/后端/Nginx前端）。
- 你在宿主机浏览器直接访问：`http://192.168.116.131/`。

## 一、已准备好的自动化脚本

- Windows 侧一键部署脚本：`scripts/vm/deploy-and-run.ps1`
- VM 侧执行脚本：`scripts/vm/run-on-vm.sh`
- Nginx 配置模板：`scripts/vm/nginx-hrdatacenter.conf`
- 后端 VM 配置：`backend/src/main/resources/application-vm.yml`

> 后端启动时会自动使用 `--spring.profiles.active=vm`，并监听 `0.0.0.0:8080`。

- `application.yml`：标准数据中心模式（真实Hive链路）
- `application-vm.yml`：VM兼容演示模式（可在无Hive集群时联调）
- 后端模式识别接口：`GET /api/system/runtime/profile`
  - 可返回 `modeName`、`activeProfiles`、`dataSourceMode`、`dataSyncEnabled` 等字段

## 二、VM 先决条件（只需准备一次）

在 `192.168.116.131` 上确认：

1. 已安装：`java8`、`nginx`、`mysql`（Hive按需）
2. 22 端口可 SSH 登录
3. 80 端口可被宿主机访问（防火墙放行）
4. MySQL 内已有库 `hr_datacenter` 与表结构（没有就先执行数据库脚本）

## 三、从 Windows 发起一键部署

在项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File "D:/HrDataCenter/scripts/vm/deploy-and-run.ps1" `
  -VmHost "192.168.116.131" `
  -VmUser "root" `
  -RootPassword "123456" `
  -MysqlPassword "123456" `
  -JwtSecret "请替换为你自己的JWT密钥"
```

如果你用密钥登录，可改为：

```powershell
  -SshKeyPath "C:/Users/你的用户名/.ssh/id_rsa"
```

> 建议：`JwtSecret` 长度至少 32 字符，过短会导致登录时 JWT 签名报错。

## 六、常见问题排查

### 1) SSH 上传失败

- 现象：`Permission denied (publickey...)`
- 处理：给 Windows 配置 SSH 私钥或启用密码登录，并确认账号有写入 `/opt/hrdatacenter` 权限

### 2) 页面可开但接口 502

- 检查 VM 后端进程：
  - `ps -ef | grep HrDataCenter`
- 查看日志：
  - `tail -n 200 /opt/hrdatacenter/logs/backend.log`

### 3) 页面 404

- 检查 Nginx 是否读取到 `hrdatacenter.conf`：
  - `nginx -t`
  - `systemctl restart nginx`
- 检查前端资源目录：
  - `/opt/hrdatacenter/frontend-dist/index.html` 是否存在

### 4) 后端启动但查不到数据

- 检查 VM MySQL 是否有数据：
  - `mysql -uroot -p -D hr_datacenter -e "show tables;"`
- 检查 `application-vm.yml` 的数据库主机与端口环境变量是否正确
