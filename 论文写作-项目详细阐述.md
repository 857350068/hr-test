## 摘要（可直接改写后使用）

本项目面向企业人力资源数字化管理场景，围绕“业务数据沉淀 + 数据分析决策 + 风险预警治理”构建一套人力资源数据中心系统。系统采用前后端分离架构：后端基于 Spring Boot 与 MyBatis-Plus 实现业务服务、权限控制与数据治理，前端基于 Vue3、Element Plus 与 ECharts 实现管理端交互与可视化。数据层采用“双数据源”策略：MySQL 承载业务事务数据，Hive 作为分析型数据源（在虚拟机兼容演示模式中可退化为 MySQL 兼容链路），配合定时同步任务完成多维分析与报表生成。项目覆盖员工、考勤、请假、绩效、薪酬、培训、招聘等业务模块，并在此基础上实现组织效能分析、人才梯队分析、薪酬分析以及流失/缺口/成本预警。系统支持 JWT 鉴权、角色权限控制、报表导出与分享、消息通知、运行模式识别等工程能力。经联调与冒烟验证，系统在虚拟机集群主节点环境下可稳定部署、启动并完成核心业务闭环，具备较好的实用性与可扩展性。

**关键词**：人力资源数据中心；Spring Boot；Vue3；MySQL；Hive；预警分析；数据治理

## 1. 研究背景与意义

### 1.1 背景

传统人力资源系统普遍存在以下问题：

- 业务系统割裂，员工、考勤、绩效、薪酬等数据分散在不同模块或系统，难以形成统一口径。
- 管理层决策依赖经验，缺少可视化、可追溯的分析支撑。
- 风险识别滞后，尤其在员工流失、人才缺口、成本超支等问题上缺乏预警机制。
- 仅依赖关系型数据库难以支撑多维统计、历史趋势与分析查询。

### 1.2 课题意义

本项目通过“业务系统 + 数据分析 + 预警治理”的一体化建设，实现：

- 将人力资源核心业务数据统一管理，降低数据孤岛问题。
- 通过分析看板、专题分析、报表中心提升管理透明度和决策效率。
- 通过规则 + 模型联合机制构建风险预警能力。
- 建立可部署、可验证、可演示的工程化原型，为后续企业级扩展提供基础。

## 2. 研究目标与需求分析

### 2.1 总体目标

- 构建可用的人力资源数据中心管理平台。
- 覆盖核心业务流程（员工、考勤、请假、绩效、薪酬、培训、招聘）。
- 提供面向管理决策的分析模块与预警模块。
- 支持角色权限、数据导出、消息通知、操作日志等治理能力。
- 支持虚拟机环境部署与主节点 IP 访问。

### 2.2 功能性需求

1. **认证与权限**：登录、注册、JWT、角色鉴权、路由权限控制。  
2. **业务数据管理**：员工档案、考勤记录、请假审批、绩效目标与评估、薪资发放与调整、培训课程与报名、招聘计划。  
3. **数据分析**：组织效能、人才梯队、薪酬分析三大专题。  
4. **风险预警**：流失风险、人才缺口、人力成本超支。  
5. **治理能力**：规则管理、模型管理、报表任务、执行日志、分享日志、操作日志、消息中心、收藏中心。  
6. **部署与运行**：支持开发模式与 VM 兼容演示模式，支持 Nginx 反代统一访问。

### 2.3 非功能性需求

- 可维护性：分层架构、模块化 API。
- 可靠性：统一异常处理、服务可重启、日志可追踪。
- 安全性：Token 鉴权、角色控制、敏感接口保护。
- 可扩展性：规则模型可配置、报表任务可调度、数据同步可开关。

## 3. 理论基础与方法路线

### 3.1 理论基础

1. **数据仓库思想**：面向分析场景进行主题化数据组织，强调历史数据沉淀与多维统计。  
2. **ETL/数据同步思想**：将业务层数据经过抽取、清洗、转换后用于分析层。  
3. **OLAP 分析思想**：支持按部门、岗位、时间等维度进行切片、汇总和趋势分析。  
4. **风险预警思想**：以阈值规则（Rule）与模型参数（Model）进行风险评分与告警输出。  
5. **RBAC 权限思想**：以角色为核心进行菜单与接口授权控制。

### 3.2 方法路线

- 以业务流程完整性为基础，先保证“数据有来源”。
- 在业务数据基础上构建分析接口，统一返回口径。
- 通过规则与模型管理实现可配置预警。
- 通过报表导出、分享与日志实现结果可留痕。
- 通过部署脚本实现“可一键部署 + 可复现实验”。

## 4. 系统总体架构设计

### 4.1 架构概览

系统采用前后端分离架构：

- **前端**：Vue3 + Element Plus + ECharts，负责页面交互与可视化展示。
- **后端**：Spring Boot + MyBatis-Plus，负责业务逻辑、鉴权、调度、导出。
- **数据层**：MySQL（业务数据）+ Hive（分析数据源；VM 模式可兼容映射到 MySQL）。
- **部署层**：Nginx 统一入口，反向代理 `/api` 到后端服务。

### 4.2 后端分层

- `controller`：REST 接口层（认证、业务、分析、预警、报表、消息等）。
- `service`：业务逻辑层（含 `service/analysis`、`service/warning`、`service/sync`）。
- `mapper/mysql`：数据访问层（MyBatis-Plus Mapper）。
- `entity/dto`：实体与请求对象。
- `config`：安全、数据源、MyBatis-Plus、WebSocket、调度等配置。
- `util/common/exception`：JWT、导出、统一响应、异常处理。

### 4.3 前端层次

- `views`：按业务域组织页面。
- `api`：按模块封装接口调用。
- `router`：路由定义 + 导航守卫。
- `utils`：Excel 导出等工具。
- `components`：通用组件复用。

## 5. 技术选型与实现依据

### 5.1 后端选型

- **Spring Boot 2.7.x**：快速构建服务与统一配置。
- **Spring Security + JWT**：实现无状态认证授权。
- **MyBatis-Plus**：减少 CRUD 模板代码，提高开发效率。
- **HikariCP**：连接池管理，提升稳定性。
- **WebSocket/STOMP**：支持消息推送场景。
- **调度任务**：支持数据同步与报表任务。

### 5.2 前端选型

- **Vue3 + Router**：组件化开发与路由管理。
- **Element Plus**：管理系统 UI 组件快速构建。
- **Axios 封装**：统一请求头、异常处理、401 重定向。
- **ECharts**：多维图表可视化展示。
- **Vite**：高效开发构建与代理联调。

### 5.3 数据源策略

- MySQL 用于事务性业务数据与管理数据。
- Hive（或 VM 兼容模式下的替代链路）用于分析型查询与报表抽取。
- 该策略兼顾了 CRUD 性能与分析扩展能力。

## 6. 数据架构与表设计说明

### 6.1 关键业务表（MySQL）

- 用户与权限：`sys_user`、`sys_role`
- 消息与日志：`sys_message`、`sys_operation_log`
- 员工主数据：`employee`
- 业务事实：`attendance`、`emp_leave`、`performance_goal`、`performance_evaluation`、`salary_payment`、`salary_adjustment`、`training_course`、`training_enrollment`、`recruitment_plan`
- 治理与配置：`analysis_rule`、`warning_model`、`data_category`
- 报表中心：`report_task`、`report_execution_log`、`report_share_log`
- 个性化能力：`user_favorite`

### 6.2 分析数据（Hive/兼容链路）

系统分析服务中采用主题化查询，例如组织维度、薪酬维度、风险维度，按部门、岗位、时间周期进行汇总，形成可视化数据源。

### 6.3 数据质量与补数

为解决演示环境常见 `NoData`，项目提供补数脚本：

- `database/1mysql/insert_data.sql`  
该脚本可批量生成员工、考勤、薪资、绩效等数据，保证看板、分析、预警页有可展示样本。

## 7. 核心模块设计与实现

### 7.1 认证与权限模块

- 登录接口签发 JWT，包含 `userId/username/roleCode`。
- 过滤器解析 `Authorization`，注入 Spring Security 上下文。
- 接口层采用 `@PreAuthorize` 进行角色约束。
- 前端路由与菜单采用 `meta.roles` + `userInfo.roleCode` 双重控制。

### 7.2 业务数据源模块

围绕 HR 核心对象构建完整管理链路：

- 员工管理：档案分页查询、详情、统计。
- 考勤管理：上班/下班打卡、考勤列表、统计。
- 请假管理：申请、审批、待办。
- 绩效管理：目标设定、考核记录。
- 薪酬管理：发放、调整、统计。
- 培训管理：课程、报名关系。
- 招聘管理：招聘计划与状态管理。

### 7.3 分析与预警模块

- 组织效能分析：部门效率、结构、人员配置、组织健康度。
- 人才梯队分析：储备、继任、能力、健康度。
- 薪酬分析：结构、竞争力、成本、优化建议。
- 预警分析：
  - 员工流失预警
  - 人才缺口预警
  - 人力成本超支预警
- 预警逻辑由规则管理与模型管理支撑，便于调参与迭代。

### 7.4 报表中心与消息协同

- 报表任务支持增删改查、定时执行、手动导出。
- 报表执行与分享日志可回溯。
- 支持报表分享消息通知。
- 前端支持导出 CSV/Excel（多 Sheet）。

### 7.5 数据治理模块

- 数据分类管理：实现业务数据分类组织。
- 规则管理：阈值规则配置与生效控制。
- 模型管理：模型版本、权重、准确率维护。
- 操作日志：治理动作留痕，便于审计与答辩展示。

## 8. 关键流程说明（论文可画时序图）

### 8.1 登录流程

1. 前端提交用户名密码到 `/api/auth/login`。  
2. 后端校验用户状态与密码。  
3. 生成 JWT 返回前端。  
4. 前端保存 `token/userInfo`，后续请求自动携带 `Authorization`。  
5. 路由守卫按登录态与角色放行或拦截。

### 8.2 分析页面加载流程

1. 页面初始化并发请求分析接口。  
2. 后端从分析数据源查询并聚合。  
3. 返回统一 `{code,message,data}`。  
4. 前端将数据映射为卡片、表格、图表。  
5. 用户可筛选、收藏、导出结果。

### 8.3 报表导出流程

1. 用户触发导出接口。  
2. 后端按报表类型查询数据并生成 CSV。  
3. 文件落盘并记录执行日志。  
4. 前端下载文件，或进行分享通知。  
5. 运维可通过执行日志追踪导出状态。

## 9. 部署方案与运行模式

### 9.1 本地开发模式

- 前端 `npm run dev`，Vite 代理 `/api` 到后端。
- 后端 `mvn spring-boot:run` 启动。

### 9.2 VM 集群主节点部署模式

- 使用 `scripts/vm/deploy-and-run.ps1` 在宿主机发起部署。
- 上传后端 jar、前端 dist、数据库脚本到主节点。
- VM 侧 `scripts/vm/run-on-vm.sh` 执行覆盖部署、可选初始化、启动后端、重启 Nginx。
- 宿主机通过主节点 IP 访问：`http://<VM-IP>/`。

### 9.3 运行模式口径（答辩重点）

- **标准模式**：真实 Hive 分析链路。
- **VM 兼容模式**：在无完整 Hive 条件下使用兼容数据源保证演示闭环。
- `GET /api/system/runtime/profile` 可输出当前模式与数据源信息。

## 11. 项目中已识别并修复的典型工程问题（可写“问题与改进”）

1. **数据库结构与代码版本不一致**：如 `sys_user.role_code` 缺失导致登录失败。  
2. **补丁脚本兼容性问题**：`ADD COLUMN IF NOT EXISTS` 在目标 MySQL 版本不兼容，改为动态检查再执行。  
3. **VM 初始化连接问题**：脚本默认 socket 导致 MySQL 连接失败，改为显式 TCP (`-h/-P`)。  
4. **报表导出目录权限问题**：路径权限导致导出失败，改为可写目录并保证创建/权限。  
5. **消息广播约束冲突**：`receiver_id` 非空导致广播写库失败，改为按用户逐条落库。  
6. **考勤打卡空值问题**：`create_time` 为空导致插入失败，补齐创建/更新时间与默认逻辑。  
7. **统计接口参数刚性**：薪资统计强依赖 `empId`，改为可选参数支持全局统计。  
8. **JWT 运行时实现加载问题**：显式指定序列化/反序列化实现并修正依赖作用域。

> 该部分建议放在论文“系统实现中的问题与优化”章节，可体现工程实践深度。
