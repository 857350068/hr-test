## 二、数据库初始化

### 1. MySQL初始化

```bash
# 登录MySQL
mysql -u root -p

# 执行初始化脚本
source database/mysql/init.sql

# 执行补充表脚本
source database/mysql/11_create_additional_tables.sql
```

**验证数据库**：
```sql
-- 查看数据库
SHOW DATABASES;

-- 使用数据库
USE hr_datacenter;

-- 查看表
SHOW TABLES;

-- 查看数据统计
SELECT '员工表' AS 表名, COUNT(*) AS 记录数 FROM employee WHERE deleted = 0
UNION ALL
SELECT '考勤表', COUNT(*) FROM attendance WHERE deleted = 0
UNION ALL
SELECT '请假表', COUNT(*) FROM `leave` WHERE deleted = 0
UNION ALL
SELECT '绩效目标表', COUNT(*) FROM performance_goal WHERE deleted = 0
UNION ALL
SELECT '薪资调整表', COUNT(*) FROM salary_adjustment WHERE deleted = 0
UNION ALL
SELECT '培训课程表', COUNT(*) FROM training_course WHERE deleted = 0;
```

### 2. Hive初始化

**启动Hive服务**：
```bash
# 启动Hive Metastore
hive --service metastore &

# 启动Hive Server2
hive --service hiveserver2 &

# 验证Hive服务
netstat -an | grep 10000
```

**初始化Hive数据仓库**：
```bash
# 执行Hive初始化脚本
hive -f database/hive/init.sql
```

**验证Hive**：
```sql
-- 连接Hive
beeline -u jdbc:hive2://localhost:10000

-- 查看数据库
SHOW DATABASES;

-- 使用数据仓库
USE hr_datacenter_dw;

-- 查看表
SHOW TABLES;
```

## 三、后端启动

### 1. 配置文件检查

**检查application.yml**：
```yaml
# MySQL配置
spring:
  datasource:
    mysql:
      url: jdbc:mysql://localhost:3306/hr_datacenter
      username: root
      password: "123456"  # 修改为你的密码

# Hive配置
    hive:
      url: jdbc:hive2://localhost:10000/hr_datacenter_dw
      username: hive
      password: ""
```


## 七、常见问题解决

### 1. MySQL连接失败

**问题**：`Communications link failure`

**解决**：
```bash
# 检查MySQL服务
systemctl status mysql

# 启动MySQL
systemctl start mysql

# 检查端口
netstat -an | grep 3306
```

### 2. Hive连接失败

**问题**：`Could not connect to HiveServer2`

**解决**：
```bash
# 检查Hive服务
jps | grep Hive

# 重启Hive服务
hive --service hiveserver2 &

# 检查端口
netstat -an | grep 10000
```

### 3. 后端启动失败

**问题**：`Port 8080 already in use`

**解决**：
```bash
# 查找占用端口的进程
netstat -an | grep 8080

# 杀死进程
kill -9 <PID>

# 或修改端口
# application.yml: server.port: 8081
```

### 4. 前端启动失败

**问题**：`npm install`失败

**解决**：
```bash
# 清除缓存
npm cache clean --force

# 使用淘宝镜像
npm install --registry=https://registry.npmmirror.com

# 或使用cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install
```

### 5. 跨域问题

**问题**：`CORS policy`

**解决**：
- 后端已配置`@CrossOrigin(origins = "*")`
- 检查vite.config.js中的proxy配置

## 八、性能优化建议

### 1. 数据库优化

**MySQL优化**：
```sql
-- 添加索引
CREATE INDEX idx_employee_dept ON employee(department);
CREATE INDEX idx_attendance_date ON attendance(attendance_date);

-- 查询优化
EXPLAIN SELECT * FROM employee WHERE department = '技术部';
```

**Hive优化**：
```sql
-- 使用分区查询
SELECT * FROM fact_attendance WHERE year_month = '2024-01';

-- 使用ORC格式
STORED AS ORC;
```

### 2. 缓存优化

**添加Redis缓存**（可选）：
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>
```
