## 1. 项目定位

### 1.1 核心定位

构建一套**人力资源数据中心** Web 管理平台，而非传统的人力资源管理系统。

**关键区别**:
- **传统HR系统**: 侧重日常业务流程管理（入职、考勤、薪酬计算等）
- **数据中心系统**: 侧重数据整合、分析和决策支持，业务模块作为数据采集层

### 1.2 系统目标（符合任务书要求）

1. **打破数据孤岛**: 整合分散在ERP、OA、考勤机、招聘渠道等不同平台的数据
2. **提升决策效率**: 利用Hive OLAP能力，快速获取关键指标，缩短决策链路
3. **实现预测性管理**: 建立离职风险预测模型，从"事后处理"转向"事前干预"
4. **技术架构革新**: 探索SpringBoot与大数据生态组件的高效集成方案

### 1.3 系统架构

采用"**数据采集层 → 数据处理层 → 分析决策层**"的三层架构：

```
业务数据采集（员工、考勤、绩效、薪酬、培训、招聘）
    ↓
数据处理与存储（MySQL业务库 + Hive分析库 + 数据同步）
    ↓
专题分析与预警（组织效能、人才梯队、薪酬分析、流失预警）
    ↓
决策支持（报表中心、分析看板、预警提示）
```

### 1.4 功能定位

在统一门户内完成：
- **数据采集与沉淀**: 人事主数据与流程类业务管理
- **专题分析**: 组织效能、人才梯队、薪酬福利等维度分析
- **风险预警**: 员工流失、人才缺口、成本超支预警
- **决策支持**: 报表与消息治理、分析看板

面向管理员、HR、部门负责人、企业管理层等角色使用。

## 2. 干系人与角色

从 `sys_role`、`sys_user.role_code` 及前端路由 `meta.roles` 可反推：

| 角色编码 | 典型能力 |
|----------|----------|
| `ROLE_ADMIN` | 全功能、系统用户管理、规则/模型、报表等 |
| `ROLE_HR_ADMIN` | 人力资源业务、分析、预警、报表等 |
| `ROLE_MANAGER` | 部门视角管理（与前端菜单配置一致部分） |
| `ROLE_EMPLOYEE` | 个人相关功能为主 |

## 3. 功能性需求

### 3.1 认证与账户

- 用户**注册**、**登录**、**退出**。
- **JWT** 无状态鉴权；请求头携带 `Authorization: Bearer <token>`。
- **个人资料**查询与修改、**修改密码**。
- 管理员对系统用户的**增删改查**、**重置密码**。

对应实现：`AuthController`、`UserManageController`、`SecurityConfig`（放行登录/注册）、`JwtAuthenticationFilter`。

### 3.2 员工与人事主数据

- 员工**分页列表**、**详情**、**新增/修改/删除**（逻辑删除）。
- **仪表盘统计**、员工总数等聚合接口。
- **Excel 批量导入**与**导出**。

对应实现：`EmployeeController`、实体 `Employee`、`mapper`。

### 3.3 考勤

- **上班/下班打卡**、**当日考勤**、**历史列表**、**统计数据**。

对应实现：`AttendanceController`、`AttendanceService`。

### 3.4 请假

- **提交申请**、**审批**、**撤回**、**我的列表**、**待审批列表**。

对应实现：`LeaveController`、表 `emp_leave`、实体 `Leave`（`@TableName("emp_leave")`）。

### 3.5 绩效

- **绩效目标**：分页、按员工查询、增删改。
- **绩效评估**：分页、自评/上级评价/面谈记录提交、改进情况概览。

对应实现：`PerformanceGoalController`、`PerformanceEvaluationController`。

### 3.6 薪酬

- **薪资发放**：分页、新增/修改、发放操作、统计。
- **薪资调整**：申请、审批、待审批列表。

对应实现：`SalaryPaymentController`、`SalaryAdjustmentController`。

### 3.7 培训

- **培训课程**分页、增删改（含删除课程）。
- **报名**分页、报名、审批、签到、成绩提交。
- **培训效果**概览。

对应实现：`TrainingController`。

### 3.8 招聘

- 招聘计划 **CRUD** 与列表。

对应实现：`RecruitmentController`、`recruitment_plan` 表。

### 3.9 数据分类

- 分类树/列表、启用分类、子节点、详情及 **CRUD**。

对应实现：`DataCategoryController`。

### 3.10 分析与可视化

- **组织效能**：部门效率、结构、编制、健康度等维度接口。
- **人才梯队**：储备、继任、能力、健康度。
- **薪酬分析**：结构、竞争力、成本、优化建议。
- **分析结果导出**（后端提供导出能力）。

对应实现：`AnalysisController`、前端 `api/analysis.js` 与各分析视图。

### 3.11 风险预警

- **员工流失**：风险分析、部门率、总览。
- **人才缺口**：分析、结构、总览。
- **成本超支**：分析、总览。

对应实现：`WarningController`、规则与模型管理配合。

### 3.12 规则、模型与运行画像

- **分析规则** CRUD、规则**生效**切换。
- **预警模型** CRUD。
- **运行模式/环境画像**查询（便于区分标准模式与 VM 演示等）。

对应实现：`RuleController`、`ModelController`、`RuntimeProfileController`。

### 3.13 报表中心

- 报表**任务**增删改查。
- 报表**导出**、**导出文件下载**、按文件名取文件。
- 报表**分享**。
- **执行记录**列表、**重建**某次记录。
- **分享记录**列表。

对应实现：`ReportController`。

### 3.14 消息与协同

- 消息**广播/发送**（管理场景）、**列表**、**未读数**、**已读/全部已读**、**删除**。
- **审批提醒**类消息。
- WebSocket 相关映射（广播/私聊）在 `MessageController` 中声明。

对应实现：`MessageController`、实体 `Message`（表 `sys_message`）。

### 3.15 收藏与操作日志

- 收藏 **列表/新增/删除**。
- 操作日志 **列表/详情/删除/批量删除/清空**。

对应实现：`FavoriteController`、`OperationLogController`。

### 3.16 文件上传

- 单文件上传、批量上传、头像上传、删除。

对应实现：`FileUploadController`。

## 4. 非功能性需求（从实现反推）

| 类别 | 说明 |
|------|------|
| 安全 | Spring Security + JWT；除登录/注册外均需认证；支持方法级 `@PreAuthorize`（已启用） |
| 跨域 | `CorsConfigurationSource` 允许跨域，适配前后端分离 |
| 可维护性 | 分层：`controller` / `service` / `mapper` / `entity` / `dto` |
| 数据层 | MySQL 事务库 + Hive 分析库双数据源；定时同步配置项见 `application.yml` 中 `data-sync` |
| 可观测 | 操作日志、报表执行/分享日志；业务与框架日志级别可配 |

## 5. 约束与假设

- 所有业务 REST 接口在部署时位于 **统一前缀** `server.servlet.context-path`（默认 **`/api`**）之下，与前端 `axios` 的 `baseURL: '/api'` 一致。
- 分析/预警部分依赖 Hive 或项目内“兼容链路”配置；具体以运行环境 `application.yml` / `application-vm.yml` 为准。
