# 前后端交互说明（核心 API）

> 基础约定：  
> - 浏览器或 Nginx 访问时，REST 前缀为 **`/api`**（与 `server.servlet.context-path` 及前端 `axios` `baseURL` 一致）。  
> - 下列路径均为 **去掉主机后的 URI**，完整形如 `http(s)://<host>:<port>/api<path>`。  
> - 请求头（除登录/注册外）：`Authorization: Bearer <token>`，`Content-Type: application/json`（文件上传除外）。  
> - 统一响应结构见后端 `Result` 类（一般为 `code`、`message`、`data`）。

**匿名可访问**（见 `SecurityConfig`）：

- `POST /api/auth/login`
- `POST /api/auth/register`

其余接口需已登录态（JWT 有效）。
