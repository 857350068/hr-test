## A. 与《任务书》《开题报告》可能不一致时的处理

仓库内较完整的“论文口径”描述见 **`docs/02-论文与答辩材料/论文写作-项目详细阐述.md`**（第 2 章需求、第 4～7 章架构与模块）。若你的正式《任务书》《开题报告》与**当前代码**不一致，建议按下表处理（任选其一：改文档表述以贴合代码，或改代码以贴合任务书）。

| 常见差异点 | 当前代码/仓库事实 | 建议 |
|------------|-------------------|------|
| SQL 目录名 | 脚本在 **`database/`**，不是 `databases/` | 统一文档与脚本中的路径；或在仓库根增加指向说明 |
| 技术栈版本 | Spring Boot **2.7.18**、Vue **3.3.x**、Java **8** | 论文中版本号与 `pom.xml`、`package.json` 对齐 |
| 数据库名 | MySQL 默认库 **`hr_datacenter`** | 任务书中若写其他库名，以部署 `application.yml` 为准并改文档 |
| 请假表名 | 物理表 **`emp_leave`**（避免 MySQL 保留字 `leave`） | 论文 ER 图与表名说明用 `emp_leave` |
| 匿名接口范围 | 仅 **`/auth/login`**、**`/auth/register`** 匿名 | 若任务书写“免登录接口列表”，与此保持一致 |
| Hive 与演示模式 | 配置中存在 **Hive** 数据源；VM 下可能有兼容/降级链路 | 答辩时明确“标准模式 vs VM 模式”的数据来源（见 `project.runtime` 与 `RuntimeProfileController`） |
| 初始化脚本破坏性 | `hr_datacenter_mysql_init.sql` 含 **DROP TABLE** | 生产禁用直接覆盖；测试库也需先备份 |

若你提供任务书/开题报告的 **Word 或 PDF**，可将其中“功能列表、技术路线、数据库名”三节与 **`docs/01-逆向工程与技术规格/`** 下的 **`需求分析.md`**、**`系统设计.md`**、**`数据库设计.md`** 做逐条 diff，把差异登记到本节表格续行即可。

## B. 虚拟机集群上替换测试数据库（MySQL）

### B.1 风险说明

- `hr_datacenter_mysql_init.sql` 会 **DROP** 表并重建，**数据不可恢复**（除非已备份）。
- 若远程存在**同名库** `hr_datacenter`，导入脚本会覆盖其中对象。
- 密码出现在命令行可能被同机用户用 `ps` 看到；生产环境建议在服务器上使用 **`~/.my.cnf`** 或受限账号。

### B.2 推荐执行顺序（MySQL）

在 **`database/`** 目录内，按依赖与语义建议顺序（可按实验需要裁剪）：

1. `hr_datacenter_mysql_init.sql` — 建库建表与基础种子数据  
2. `mysql_patch_20260416.sql` — 若有结构补丁  
3. `update_user_passwords.sql` — 若需统一演示账号密码  
4. `1mysql/insert_data.sql` — 中小批量补数  
5. `1mysql/insert_large_data.sql` — 大体量补数（耗时更长）

**Hive**（需在集群上已部署 HiveServer2，且你有权限）：

- `hr_datacenter_hive_init.sql`  
- `2hive/insert_data.sql` / `2hive/insert_large_data.sql`  

Hive 不建议与 MySQL 用同一套 PowerShell 盲目管道导入；应在节点上用 `beeline` 执行，具体 URL 与库名以集群为准。

### B.3 使用自动化脚本（Windows 宿主机）

1. 安装 **OpenSSH 客户端**（Windows 可选功能），保证 `ssh`、`scp` 在 PATH 中。  
2. 将仓库内 **`scripts/deploy_db.ps1`** 复制到可执行位置，编辑脚本顶部 **配置变量**（远程 IP、用户、密钥、MySQL 账号等）。  
3. 先 **`Get-Help`** 或阅读脚本内注释，确认 `-Confirm:$false` 等危险开关。  
4. 在 **PowerShell** 中执行（示例）：

```powershell
cd D:\HrDataCenter
Set-ExecutionPolicy -Scope Process Bypass
.\scripts\deploy_db.ps1
```

5. 执行后在远程用应用配置中的账号验证：`mysql -e "USE hr_datacenter; SHOW TABLES;"`

### B.4 执行后应用侧

- 若库地址/库名未变，仅重建数据：重启 Spring Boot 或等待连接池重连即可。  
- 若修改了库名或 IP：同步修改 **`application.yml` / `application-vm.yml`** 并重新打包部署后端。

## C. 文档与代码同步约定

任一以下变更必须同步更新 `docs/01-逆向工程与技术规格/` 下四份主文档：

- 新增或删除 Controller 级 API  
- 表结构或实体 `@TableName` 变更  
- 前端 `baseURL` 或网关路径变更  
- 角色编码或鉴权策略变更  

主文档路径（均在 `docs/01-逆向工程与技术规格/`）：

- `需求分析.md`  
- `系统设计.md`  
- `数据库设计.md`  
- `前后端交互.md`
