# 第3章 系统需求分析

## 3.1 需求分析概述

### 3.1.1 需求分析目的

需求分析是软件开发过程中的关键环节，其目的是明确系统需要实现的功能和满足的性能指标，为后续的系统设计和实现提供依据。本系统的需求分析主要基于以下考虑：

1. **明确系统定位**：本系统是一个人力资源数据中心，而非传统的人力资源管理系统，需要明确两者的区别和本系统的核心价值。

2. **满足任务书要求**：需求分析需要完全覆盖任务书要求的所有功能点，包括前端界面功能、用户功能、后台管理员功能。

3. **体现数据中心价值**：需求分析需要突出数据整合、深度分析、智能决策等数据中心的核心能力。

4. **指导系统设计**：需求分析的结果将直接指导后续的系统架构设计、功能模块设计和数据库设计。

### 3.1.2 需求分析方法

本系统采用以下需求分析方法：

1. **任务书分析法**：详细分析任务书中的功能要求，确保系统实现所有规定功能。

2. **用户角色分析法**：根据不同用户角色（管理员、HR管理员、部门负责人、普通员工）分析各自的功能需求。

3. **业务流程分析法**：分析人力资源管理的业务流程，识别关键业务节点和数据流转过程。

4. **非功能需求分析法**：从性能、安全、可靠性、可扩展性、易用性等方面分析系统的非功能需求。

## 3.2 系统功能需求分析

### 3.2.1 用户角色分析

本系统涉及四类用户角色，每类角色具有不同的权限和功能需求：

#### 1. 系统管理员（ROLE_ADMIN）

**角色定位**：系统的最高权限用户，负责系统的整体管理和维护。

**主要职责**：
- 管理所有系统用户，分配用户角色和权限
- 管理系统的基础配置，包括数据分类、分析规则、预警模型
- 监控系统运行状态，查看操作日志
- 生成和管理企业级报表
- 维护系统安全和数据完整性

**权限范围**：拥有系统的所有权限，可以访问所有功能模块。

#### 2. HR管理员（ROLE_HR_ADMIN）

**角色定位**：人力资源部门的专业人员，负责人力资源业务的管理和分析。

**主要职责**：
- 管理员工档案、薪酬、绩效、培训等人力资源数据
- 查看和分析组织效能、人才梯队、薪酬福利等专题数据
- 查看员工流失、人才缺口、成本超支等预警信息
- 跟踪培训效果和绩效改进情况
- 生成和分享人力资源分析报表

**权限范围**：可以访问人力资源业务模块、数据分析模块、预警模块、报表模块。

#### 3. 部门负责人（ROLE_MANAGER）

**角色定位**：各部门的管理者，负责本部门的人力资源管理。

**主要职责**：
- 查看本部门员工的基本信息和绩效情况
- 查看本部门员工的培训参与和效果情况
- 查看本部门相关的人力资源分析数据
- 查看本部门相关的预警信息
- 管理本部门的招聘需求

**权限范围**：只能访问本部门相关的数据和功能。

#### 4. 普通员工（ROLE_EMPLOYEE）

**角色定位**：企业的普通员工，使用系统的个人功能。

**主要职责**：
- 管理个人信息，修改密码
- 查看个人的考勤、请假、绩效、薪酬记录
- 参与培训报名，查看培训成绩
- 收藏感兴趣的分析报表和预警报告

**权限范围**：只能访问个人相关的数据和功能。

### 3.2.2 前端界面功能需求

根据任务书要求，前端界面需要实现以下功能：

#### 3.2.2.1 数据分类展示需求

**功能描述**：展示人力资源数据的分类信息，帮助用户快速定位所需数据。

**具体需求**：
1. **分类体系展示**：
   - 展示组织效能类、人才梯队类、薪酬福利类、绩效管理类等数据分类
   - 支持多级分类展示，如组织效能类下包含部门效率、组织架构、人员配置等子分类
   - 分类以树形结构或卡片形式展示，便于用户浏览

2. **分类导航功能**：
   - 用户点击分类后，跳转到对应的数据展示页面
   - 支持分类的快速切换和筛选
   - 记录用户的常用分类，提供快捷访问

3. **分类管理功能**（管理员）：
   - 管理员可以新增、修改、删除数据分类
   - 支持分类的启用和禁用
   - 支持分类的排序和层级调整

**界面要求**：
- 分类展示清晰，层次分明
- 支持图标和颜色区分不同分类
- 响应式设计，适配不同屏幕尺寸

#### 3.2.2.2 数据显示与筛选需求

**功能描述**：默认显示所有人力资源核心分析数据，支持按条件筛选查看。

**具体需求**：
1. **数据默认展示**：
   - 默认显示所有人力资源核心分析数据
   - 展示内容包括：数据维度、统计周期、核心指标数值、趋势图表等
   - 数据以卡片、表格、图表等多种形式展示

2. **按分类筛选**：
   - 用户可以点击数据分类，查看指定分类的人力资源分析数据
   - 筛选后只显示该分类相关的数据
   - 支持多分类同时筛选

3. **按维度筛选**：
   - 支持按部门筛选：选择特定部门查看该部门的数据
   - 支持按岗位筛选：选择特定岗位查看该岗位的数据
   - 支持按时间维度筛选：选择特定时间段查看数据
   - 支持多维度组合筛选

4. **数据对比功能**：
   - 支持不同时间段的数据对比
   - 支持不同部门的数据对比
   - 对比结果以图表形式展示

5. **数据导出功能**：
   - 支持将筛选后的数据导出为Excel格式
   - 支持导出图表为图片格式
   - 导出文件命名规范，便于识别

**界面要求**：
- 筛选条件设置便捷，支持快捷选择
- 数据展示直观，支持多种图表类型
- 支持数据的钻取和展开

#### 3.2.2.3 模糊查询需求

**功能描述**：支持根据关键词进行模糊查询，快速定位所需数据。

**具体需求**：
1. **查询条件**：
   - 支持按部门名称模糊查询
   - 支持按岗位名称模糊查询
   - 支持按统计周期查询（如"2024年第一季度"）
   - 支持按员工编号查询
   - 支持多条件组合查询

2. **查询结果展示**：
   - 查询结果以列表形式展示
   - 高亮显示匹配的关键词
   - 显示结果的统计数量
   - 支持结果的排序和分页

3. **查询历史**：
   - 记录用户的查询历史
   - 支持快速重复查询
   - 支持查询条件的保存

**界面要求**：
- 查询输入框支持自动补全
- 查询响应速度快，实时显示结果
- 支持查询条件的清除和重置

#### 3.2.2.4 用户登录与注册需求

**功能描述**：支持多角色用户的登录与注册，实现身份认证。

**具体需求**：
1. **用户注册功能**：
   - 支持HR管理员、部门负责人、企业管理层等不同角色的注册
   - 注册时需要填写用户名、密码、真实姓名、手机号、邮箱等基本信息
   - 密码需要进行强度校验，要求包含字母和数字，长度不少于6位
   - 注册信息需要进行格式校验，如手机号、邮箱格式校验
   - 注册成功后自动跳转到登录页面

2. **用户登录功能**：
   - 用户输入用户名和密码进行登录
   - 支持记住密码功能
   - 登录失败时给出明确的错误提示
   - 登录成功后根据用户角色跳转到对应的主页面
   - 生成JWT Token，用于后续的身份认证

3. **身份认证功能**：
   - 每次请求都需要携带JWT Token
   - Token过期后需要重新登录
   - 支持Token的自动刷新
   - 未登录用户访问受保护资源时跳转到登录页面

4. **密码安全功能**：
   - 密码使用BCrypt算法加密存储
   - 支持密码修改功能，需要验证原密码
   - 支持密码找回功能（通过邮箱或手机号）

**界面要求**：
- 登录和注册界面简洁美观
- 表单校验提示清晰
- 支持响应式设计，适配移动端

#### 3.2.2.5 分析看板需求

**功能描述**：用户登录后可查看所属权限范围内的人力资源数据看板，展示关键指标及预警信息。

**具体需求**：
1. **关键指标展示**：
   - 展示员工总数、部门数量、平均薪资等关键指标
   - 展示人才流转率、人力成本占比、培训投入产出比等核心指标
   - 指标以卡片形式展示，支持点击查看详情
   - 支持指标的同比和环比分析

2. **趋势图表展示**：
   - 展示关键指标的历史趋势图
   - 支持按日、周、月、年查看趋势
   - 支持多个指标的趋势对比
   - 图表支持交互，如缩放、数据点提示

3. **预警信息展示**：
   - 展示最新的预警信息，如员工流失预警、人才缺口预警
   - 预警信息按严重程度分级显示（高、中、低）
   - 支持预警信息的快速处理和标记
   - 点击预警信息可查看详情

4. **权限范围控制**：
   - 不同角色用户看到不同范围的数据
   - HR管理员可以看到全企业的数据
   - 部门负责人只能看到本部门的数据
   - 普通员工只能看到个人的数据

5. **看板定制功能**：
   - 支持用户自定义看板布局
   - 支持选择显示的指标和图表
   - 支持看板配置的保存和恢复

**界面要求**：
- 看板布局合理，信息层次清晰
- 图表美观，交互流畅
- 支持数据的实时刷新

#### 3.2.2.6 收藏功能需求

**功能描述**：用户登录后可以收藏常用的人力资源分析报表、预警报告、人才档案等内容。

**具体需求**：
1. **收藏添加功能**：
   - 用户可以在分析报表页面点击收藏按钮
   - 用户可以在预警报告页面点击收藏按钮
   - 用户可以在人才档案页面点击收藏按钮
   - 收藏时可以添加备注和标签

2. **收藏管理功能**：
   - 用户可以查看自己的收藏列表
   - 支持按收藏类型筛选（报表、预警、档案等）
   - 支持按标签筛选
   - 支持收藏的排序（按时间、按名称）

3. **收藏操作功能**：
   - 支持取消收藏
   - 支持修改收藏的备注和标签
   - 支持收藏的批量操作
   - 支持收藏的分享

4. **收藏提醒功能**：
   - 收藏的报表有更新时提醒用户
   - 收藏的预警有新情况时提醒用户
   - 支持提醒的关闭和设置

**界面要求**：
- 收藏按钮位置明显，操作便捷
- 收藏列表展示清晰，支持快速访问
- 支持收藏的分组和标签管理

### 3.2.3 用户功能需求

根据任务书要求，用户功能需要实现以下内容：

#### 3.2.3.1 个人中心功能需求

**功能描述**：用户可以管理自己的个人信息和账户设置。

**具体需求**：
1. **个人信息查看**：
   - 查看用户名、真实姓名、角色、部门等基本信息
   - 查看手机号、邮箱等联系方式
   - 查看最后登录时间、账户创建时间等

2. **个人信息修改**：
   - 修改真实姓名
   - 修改手机号（需要验证）
   - 修改邮箱（需要验证）
   - 上传和修改头像

3. **密码修改功能**：
   - 输入原密码进行验证
   - 输入新密码，需要进行强度校验
   - 确认新密码，需要两次输入一致
   - 修改成功后需要重新登录

4. **权限信息完善**：
   - 部门负责人可以设置负责的部门
   - 可以设置查看权限范围
   - 可以设置通知偏好

**界面要求**：
- 个人中心界面简洁清晰
- 表单校验提示明确
- 操作流程简单易懂

#### 3.2.3.2 数据查看与分析需求

**功能描述**：查看组织效能、人才梯队、薪酬福利等维度的分析数据，支持筛选和导出。

**具体需求**：
1. **组织效能分析数据查看**：
   - 查看部门效率分析数据：人均产出、工作饱和度、协作效率
   - 查看组织架构分析数据：层级结构、管理幅度、组织扁平化程度
   - 查看人员配置分析数据：编制执行率、人岗匹配度、人员结构合理性
   - 查看组织健康度数据：员工满意度、团队稳定性、组织活力指数

2. **人才梯队建设数据查看**：
   - 查看人才储备数据：关键岗位人才储备率、高潜人才识别
   - 查看继任计划数据：关键岗位继任者准备度、继任计划覆盖率
   - 查看能力评估数据：能力模型、能力差距分析、发展建议
   - 查看梯队健康度数据：梯队完整性、梯队成熟度、梯队流动性

3. **薪酬福利分析数据查看**：
   - 查看薪酬结构数据：内部公平性、外部竞争力、薪酬结构合理性
   - 查看竞争力分析数据：市场对标、分位值分析、竞争力指数
   - 查看成本分析数据：人力成本占比、成本结构、成本趋势
   - 查看优化建议：薪酬调整建议、成本优化方案

4. **数据筛选功能**：
   - 支持按部门筛选：选择特定部门查看该部门的分析数据
   - 支持按岗位筛选：选择特定岗位查看该岗位的分析数据
   - 支持按时间维度筛选：选择特定时间段查看数据
   - 支持多维度组合筛选

5. **数据对比功能**：
   - 支持不同时间段的数据对比
   - 支持不同部门的数据对比
   - 支持不同岗位的数据对比
   - 对比结果以图表形式展示

6. **数据导出功能**：
   - 支持将分析数据导出为Excel格式
   - 支持导出图表为图片格式
   - 支持导出PDF报告
   - 导出文件包含数据说明和分析结论

**界面要求**：
- 数据展示直观，支持多种图表类型
- 筛选条件设置便捷
- 支持数据的钻取和展开
- 导出功能操作简单

#### 3.2.3.3 预警信息查看需求

**功能描述**：查看员工流失预警、人才缺口预警、人力成本超支预警等信息。

**具体需求**：
1. **员工流失预警查看**：
   - 查看流失风险分析：离职风险预测、高风险员工识别
   - 查看部门流失率：各部门流失率对比、流失趋势分析
   - 查看预警概览：流失预警汇总、应对建议
   - 支持预警详情查看，包括预警原因、风险等级、建议措施

2. **人才缺口预警查看**：
   - 查看缺口分析：关键岗位缺口、技能缺口、数量缺口
   - 查看结构分析：缺口分布、缺口趋势、缺口影响
   - 查看预警概览：缺口预警汇总、应对建议
   - 支持预警详情查看，包括缺口原因、影响范围、补充建议

3. **人力成本超支预警查看**：
   - 查看成本分析：成本执行情况、超支风险识别
   - 查看预警概览：成本预警汇总、控制建议
   - 支持预警详情查看，包括超支原因、超支金额、控制措施

4. **预警记录查询**：
   - 支持按预警类型查询
   - 支持按预警等级查询
   - 支持按时间范围查询
   - 支持按部门查询

5. **预警处理功能**：
   - 支持预警的确认和处理
   - 支持添加处理意见和措施
   - 支持预警的关闭和归档
   - 记录预警的处理历史

**界面要求**：
- 预警信息展示醒目，按严重程度分级显示
- 支持预警的快速处理
- 预警详情信息完整
- 支持预警的统计和分析

#### 3.2.3.4 培训效果跟踪需求

**功能描述**：HR管理员可查看培训需求分析结果、培训效果量化数据，部门负责人可查看本部门员工培训参与及效果情况。

**具体需求**：
1. **培训需求分析查看**（HR管理员）：
   - 查看培训需求识别结果
   - 查看培训需求优先级排序
   - 查看培训需求与业务目标的关联
   - 支持培训需求的筛选和导出

2. **培训效果量化查看**：
   - 查看到课率统计：各部门、各课程的到课率
   - 查看考试成绩统计：平均成绩、成绩分布、及格率
   - 查看满意度调查结果：培训满意度、讲师满意度
   - 查看培训投入产出比（ROI）

3. **本部门培训情况查看**（部门负责人）：
   - 查看本部门员工的培训报名情况
   - 查看本部门员工的培训参与情况
   - 查看本部门员工的培训成绩
   - 查看本部门员工的培训反馈

4. **培训数据查询**：
   - 支持按培训课程查询
   - 支持按员工查询
   - 支持按时间范围查询
   - 支持按部门查询
