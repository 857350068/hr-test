# 毕业论文摘要

## 摘要

随着企业人力资源数据规模的快速增长和数字化转型的深入推进，传统的人力资源管理系统在处理海量历史数据、多维度关联分析和预测性决策方面存在明显不足。数据分散、分析能力弱、决策缺乏数据支撑等问题，严重制约了企业人力资源管理的科学化水平。为此，本文设计并实现了一个基于SpringBoot和Hive的人力资源数据中心系统。

系统采用"技术支撑-数据处理-功能应用"三位一体的分层架构。底层技术支撑层使用SpringBoot 2.7.18和MyBatis-Plus 3.5.3.1构建后端服务，Vue 3和Element Plus实现前端交互，MySQL 8.0存储业务数据，Hive 3.1.3存储分析数据。中间数据处理层通过定时同步任务将MySQL业务数据同步至Hive数据仓库，利用Hive的OLAP能力进行海量数据分析，结合分析规则和预警模型实现实时预警计算。上层功能应用层围绕人力资源管理全流程，实现数据采集与沉淀、专题分析、预警分析和决策支持等功能。顶层用户权限层支持管理员、HR管理员、部门负责人和普通员工四种角色，实现菜单级和方法级的双重权限控制。

系统功能按用户角色划分为三个层次。**HR管理员和企业管理层**可查看组织效能分析、人才梯队建设、薪酬福利分析等专题分析数据，支持按部门、岗位、时间维度筛选，实现数据的查询、对比、导出功能；查看员工流失预警、人才缺口预警、人力成本超支预警等预警信息，实现预警记录的查询和详情查看；查看培训需求分析结果、培训效果量化数据，实现数据的查询和反馈功能；查看绩效指标完成情况、考核结果分布、绩效改进计划执行进度，实现跟踪记录的查询和更新功能。**部门负责人**可查看本部门员工培训参与及效果情况，查看本部门相关的人力资源分析数据和预警信息。**后台管理员**可管理所有系统用户，分配不同角色的查看和操作权限；管理所有人力资源基础数据和分析结果数据，实现数据的增删改查功能，同步MySQL基础数据至Hive；对数据分类、分析规则、预警模型进行管理，配置Hive分析任务的执行周期；生成全企业的人力资源分析可视化报表，支持报表的定时生成、导出、分享功能。

本系统的实现打破了企业人力资源管理中的数据孤岛，通过整合多源数据、构建分析模型和提供可视化工具，实现了人力资源管理的数字化转型。系统利用Hive强大的OLAP能力，显著提升了人才流转率、人力成本占比、培训投入产出比等关键指标的获取效率，缩短了决策链路。通过建立离职风险预测模型，帮助HR从"事后处理"转向"事前干预"，提升了企业人力资源管理的数字化决策能力和管理效率，为企业人力资源管理的科学化、智能化发展提供了有力支持。

**关键词**：人力资源数据中心；SpringBoot；Hive；OLAP分析；预警模型；三位一体架构

## Abstract

With the rapid growth of enterprise human resource data scale and the deepening advancement of digital transformation, traditional human resource management systems have obvious deficiencies in processing massive historical data, multi-dimensional association analysis, and predictive decision-making. Problems such as scattered data, weak analytical capabilities, and lack of data support for decision-making seriously restrict the scientific level of enterprise human resource management. Therefore, this paper designs and implements a human resource data center system based on SpringBoot and Hive.

The system adopts a three-in-one layered architecture of "technical support - data processing - functional application". The bottom technical support layer uses SpringBoot 2.7.18 and MyBatis-Plus 3.5.3.1 to build backend services, Vue 3 and Element Plus to implement frontend interaction, MySQL 8.0 to store business data, and Hive 3.1.3 to store analytical data. The middle data processing layer synchronizes MySQL business data to the Hive data warehouse through scheduled synchronization tasks, utilizes Hive's OLAP capabilities for massive data analysis, and combines analysis rules and warning models to achieve real-time warning calculations. The upper functional application layer revolves around the entire human resource management process, implementing functions such as data collection and precipitation, thematic analysis, warning analysis, and decision support. The top user permission layer supports four roles: administrator, HR administrator, department head, and ordinary employee, achieving dual permission control at both menu and method levels.

System functions are divided into three levels according to user roles. **HR administrators and enterprise management** can view thematic analysis data such as organizational efficiency analysis, talent pipeline construction, and compensation and benefits analysis, support filtering by department, position, and time dimension, and achieve data query, comparison, and export functions; view warning information such as employee turnover warnings, talent gap warnings, and labor cost overrun warnings, and achieve query and detail viewing of warning records; view training needs analysis results and training effectiveness quantitative data, and achieve data query and feedback functions; view performance indicator completion, assessment result distribution, and performance improvement plan execution progress, and achieve query and update of tracking records. **Department heads** can view the training participation and effectiveness of employees in their department, and view department-related human resource analysis data and warning information. **Backend administrators** can manage all system users and assign viewing and operation permissions for different roles; manage all human resource basic data and analysis result data, achieve data CRUD functions, and synchronize MySQL basic data to Hive; manage data classification, analysis rules, and warning models, and configure the execution cycle of Hive analysis tasks; generate enterprise-wide human resource analysis visualization reports, supporting scheduled generation, export, and sharing of reports.

The implementation of this system breaks the data silos in enterprise human resource management, and achieves the digital transformation of human resource management by integrating multi-source data, building analysis models, and providing visualization tools. The system utilizes Hive's powerful OLAP capabilities to significantly improve the efficiency of obtaining key indicators such as talent turnover rate, labor cost ratio, and training ROI, shortening the decision-making chain. By establishing a turnover risk prediction model, it helps HR shift from "post-event handling" to "pre-event intervention", improving the digital decision-making ability and management efficiency of enterprise human resource management, and providing strong support for the scientific and intelligent development of enterprise human resource management.

**Keywords**: Human Resource Data Center; SpringBoot; Hive; OLAP Analysis; Warning Model; Three-in-One Architecture
