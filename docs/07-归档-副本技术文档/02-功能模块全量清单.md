# HrDataCenter 功能模块全量清单（代码 + UI）

## 1. 总体说明

- 本清单从“前端页面 + 前端API + 后端控制器/服务/实体 + 数据库脚本”四个维度梳理。
- 用途：用于你后续从头重构时，逐模块对照迁移，避免漏功能、漏接口、漏表结构。

## 4. UI设计还原建议（重构时）

- 先复刻路由骨架与Layout导航，再逐业务页迁移，防止权限菜单和跳转逻辑丢失。
- 每个 `views` 页面要和同名 `api` 模块一一对应，确保前后端契约稳定。
- 分析与预警页面优先恢复图表与筛选交互，再接入后端真实数据。
- 系统治理页面按“用户/日志/规则/模型/报表”顺序重建，便于权限联调。

## 5. 重构验收最小清单

- 账号登录、注册、权限控制可用。
- 员工/考勤/请假/薪资/绩效/培训/招聘 CRUD 闭环可用。
- 分析中心四大页面可加载数据并完成图表展示。
- 预警分析可查询并展示预警结果。
- 系统管理核心页面（用户、日志、规则、模型、报表）可用。
