# HrDataCenter 重构技术档案

## 1. 项目目标与范围

- 项目类型：人力资源数据中心（业务管理 + 统计分析 + 预警 + 系统管理）。
- 重构目标：在保留当前功能边界的前提下，从零重建前端、后端、数据库三层。
- 当前基线来源：`D:/HrDataCenter` 主仓库源码（已提取为本目录副本）。

## 2. 提取副本结构

- `副本/前端`：前端源码与配置（53个文件）。
- `副本/后端`：后端源码与配置（115个文件）。
- `副本/数据库`：SQL脚本（9个文件）。
- `docs/07-归档-副本技术文档`：重构文档集合（原 `副本/技术文档`）。

## 3. 技术栈与版本清单

### 3.1 前端（Vue 3）

- Node生态：`npm`（由 `package-lock.json` 锁定依赖）。
- 核心框架：`vue@^3.3.4`。
- 路由：`vue-router@^4.2.4`。
- UI组件：`element-plus@^2.3.14`、`@element-plus/icons-vue@^2.1.0`。
- 网络请求：`axios@^1.5.0`。
- 图表：`echarts@^5.4.3`。
- 导出与文件：`xlsx@^0.18.5`、`file-saver@^2.0.5`。
- 构建工具：`vite@^4.4.9`、`@vitejs/plugin-vue@^4.3.4`、`terser@^5.46.1`。

### 3.2 后端（Spring Boot）

- JDK：`Java 1.8`。
- Spring Boot：`2.7.18`。
- ORM与持久层：`mybatis-plus-boot-starter 3.5.3.1`。
- 数据库驱动：`mysql-connector-java 8.0.33`。
- 鉴权：`jjwt 0.11.5`（api/impl/jackson）。
- 数据分析相关：`hadoop-common 3.2.4`、`hive-jdbc 3.1.3`、`hive-service 3.1.3`。
- 工具库：`hutool-all 5.8.20`。
- 构建：Maven，`spring-boot-maven-plugin`，`maven-compiler-plugin 3.8.1`。

### 3.3 数据层

- 关系库：MySQL（业务主库）。
- 数仓/分析库：Hive（分析数据源）。
- 多数据源：应用层中存在 MySQL + Hive 双数据源配置。

## 4. 应用配置基线

- 服务端口：`8080`。
- 后端上下文：`/api`。
- 已脱敏文件：`副本/后端/application.yml.example`。
- 脱敏项：
  - MySQL 密码改为 `${MYSQL_PASSWORD}`。
  - JWT 密钥改为 `${JWT_SECRET}`。

## 5. 代码组织标准（重构时建议保持）

### 5.1 前端目录建议

- `src/views`：业务页面（按业务域分目录）。
- `src/api`：按业务域拆分接口模块。
- `src/router`：路由与权限控制。
- `src/components`：通用组件。
- `src/utils`：工具函数与辅助逻辑。

### 5.2 后端目录建议

- `controller`：API入口层（REST接口）。
- `service`：业务逻辑层。
- `mapper`：数据访问层（MyBatis Mapper）。
- `entity`：实体定义。
- `config`：多数据源、安全、MyBatis等配置。
- `resources`：应用配置与Mapper XML。

## 6. 启动与验证流程（重构参考）

### 6.1 数据库初始化

1. 执行 `副本/数据库/hr_datacenter_mysql_init.sql`。
2. 执行 `副本/数据库/hr_datacenter_hive_init.sql`。
3. 按需执行插数脚本（`insert_data.sql` / `insert_large_data.sql`）。

### 6.2 后端启动

1. 进入 `副本/后端`。
2. 配置环境变量（至少 `MYSQL_PASSWORD`、`JWT_SECRET`）。
3. 执行 `mvn clean package` 后启动应用。

### 6.3 前端启动

1. 进入 `副本/前端`。
2. 执行 `npm install`。
3. 执行 `npm run dev` 启动开发环境。

## 7. 重构建议策略

- 第一阶段：先搭建认证、用户、组织、员工等基础主数据模块。
- 第二阶段：补齐考勤、请假、薪资、绩效、培训、招聘主流程。
- 第三阶段：补齐分析看板、预警模型、报表中心等分析能力。
- 第四阶段：完善系统治理能力（日志、消息、规则、收藏、权限）。
